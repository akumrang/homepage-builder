## 1. 목적

이 문서는 실제 고객이 없는 상태에서, 첫 파일럿 발송 문안에 고객이 회신했다고 가정하고 접수 기록 양식이 충분한지 다시 점검한 기록이다.

현재 단계는 실제 고객 회신 접수가 아니다. 모든 학원명, 회신일, 자료 상태, 연락 방식은 가상값 또는 판정 예시다.

검토 대상:

- `docs/17_FIRST_PILOT_CUSTOMER_SEND_COPY.md`
- `docs/18_FIRST_PILOT_INTAKE_RECEIPT_RECORD_TEMPLATE.md`
- `docs/19_FIRST_PILOT_INTAKE_RECEIPT_REHEARSAL.md`
- `docs/28_FIRST_PILOT_SEND_COPY_VIRTUAL_REHEARSAL.md`

## 2. 재점검 판정

판정: `PASS WITH HOLD GATES`

접수 기록 양식은 최신 발송 문안과 연결된다.

다만 `MATERIALS_READY`와 `CUSTOMER_PUBLISHED`는 반드시 분리해서 판단해야 한다.

- `MATERIALS_READY`: 필수 정보와 공개 가능성 확인이 충분하면 가능
- `CUSTOMER_PUBLISHED`: 사진/로고 사용 승인, 고객 최종 확인, 샘플 문구 제거까지 확인해야 가능

이번 리허설의 핵심 결론은 다음이다.

```text
가상 회신 접수는 초안 제작 가능 여부를 판단하는 절차이지,
실제 고객 게시 승인 절차가 아니다.
```

## 3. 가상 회신 시나리오

### 3.1 시나리오 A: 정상 회신, 일부 게시 전 보류

가정:

- 가상 학원명은 `[가상] 온빛국어학원`이다.
- 학원명, 대표 과목, 대상 학년, 공개 연락처, 주소, 상담 방식, 커리큘럼은 제출되었다.
- 로고 파일은 없지만 텍스트 로고 사용은 허용했다.
- 대표 사진 후보는 제출되었지만 사진 사용 동의는 아직 명확하지 않다.
- 강사 경력은 증빙 자료 없이 수업 방식 중심 공개를 선택했다.
- 학생 이름, 성적, 학부모 연락처, 결제 정보, 시험지 원본은 제출되지 않았다.

판정:

| 항목 | 결과 |
|---|---|
| 개인정보/민감 자료 1차 차단 | `PASS` |
| 필수 자료 | `READY` |
| 권장 자료 | `SKIP_OK` |
| 로고 | 텍스트 로고 fallback 가능 |
| 대표 사진 | 후보 있음, 사용 동의 확인 전 게시 보류 |
| 초안 제작 | 가능 |
| `MATERIALS_READY` | 가능 |
| `CUSTOMER_PUBLISHED` | 불가 |

결론:

```text
WAITING_FOR_MATERIALS -> MATERIALS_READY 가능
CUSTOMER_PUBLISHED 전환은 보류
```

### 3.2 시나리오 B: 민감 자료 포함 회신

가정:

- 가상 회신 자료에 학생 이름이 포함된 수업 사례가 들어 있다.
- 사진 후보 중 일부에 학생 얼굴 또는 이름표가 보인다.
- 학부모 연락처 목록 또는 상담 기록 일부가 포함된 파일이 함께 왔다고 가정한다.

판정:

| 항목 | 결과 |
|---|---|
| 학생 이름 | `있음` |
| 사진 속 학생 얼굴 | `있음` |
| 사진 속 이름표 | `있음` |
| 학부모 연락처 목록 | `있음` |
| 개인정보/민감 자료 1차 차단 | `HOLD` |
| 해당 자료 사용 | 불가 |
| 재제출 요청 | 필요 |
| `MATERIALS_READY` | 불가 |
| `CUSTOMER_PUBLISHED` | 불가 |

결론:

```text
WAITING_FOR_MATERIALS 유지
개인정보/민감 자료 제거 또는 재제출 전에는 초안 제작 자료로 사용하지 않음
```

## 4. 접수 기록 양식 적합성

`docs/18_FIRST_PILOT_INTAKE_RECEIPT_RECORD_TEMPLATE.md`는 이번 재점검 기준으로 다음 항목을 충분히 담고 있다.

| 필요 판단 | 양식 내 위치 | 판정 |
|---|---|---|
| 개인정보/민감 자료 차단 | 4. 개인정보와 민감 자료 1차 차단 | 충분 |
| 필수 자료 충족 여부 | 5. 필수 자료 접수 체크 | 충분 |
| 권장 자료와 생략 가능 항목 | 6. 권장 자료 접수 체크 | 충분 |
| 로고 파일 없음과 텍스트 로고 fallback | 7.1 로고 | 충분 |
| 사진 후보와 사용 동의 미확인 분리 | 7.2 대표 사진 | 충분 |
| 콘텐츠 초안 반영 가능성 | 8. 콘텐츠 반영 판정 | 충분 |
| 제작 상태 전환 판단 | 9. 제작 상태 판정 | 충분 |
| 부족 자료 재요청 | 10. 부족 자료 재요청 목록 | 충분 |
| 실제 고객 정보 비저장 | 11. 내부 기록 원칙 | 충분 |

보완 필요는 크지 않다.

다만 향후 내부 입력 화면을 만들 때는 다음을 UI에서 분리해야 한다.

- `초안 제작 가능`
- `고객 확인 가능`
- `고객 게시 가능`

이 셋을 하나의 체크로 합치면 운영 위험이 커진다.

## 5. 기존 가상 판정 문서와의 관계

`docs/19_FIRST_PILOT_INTAKE_RECEIPT_REHEARSAL.md`는 여전히 유효하다.

이번 재점검은 다음을 추가로 확인했다.

- 최신 발송 문안의 고객 발송 범위와 접수 양식이 연결되는지
- 실제 고객이 없는 현재 단계에서 가상 정보만 사용하고 있는지
- 민감 자료가 섞인 회신 시나리오에서 `HOLD`가 명확히 작동하는지
- `MATERIALS_READY`와 `CUSTOMER_PUBLISHED`를 분리해서 판단하는지

따라서 `docs/19`는 기본 가상 판정 예시, 이 문서는 최신 발송 패키지 기준 재점검 기록으로 둔다.

## 6. 재요청 문안 점검

### 6.1 시나리오 A 재요청 문안

```text
자료 확인 감사합니다.

홈페이지 초안 제작에 필요한 기본 자료는 충분히 확인되었습니다.
초안 제작은 진행할 수 있으나, 실제 게시 전 아래 항목만 추가 확인 부탁드립니다.

1. 제출해주신 대표 사진을 홈페이지에 사용해도 되는지 확인 부탁드립니다.
2. 초안 확인 후 최종 게시 승인 여부를 알려주세요.
3. 주차 안내가 필요하면 가능한 범위만 알려주세요. 없으면 생략하겠습니다.

베틀 시스템은 위 항목 확인 전에는 일반 공개 게시로 전환하지 않겠습니다.
```

판정: `PASS`

이 문안은 초안 제작 가능성과 게시 보류를 분리해서 설명한다.

### 6.2 시나리오 B 재요청 문안

```text
자료 확인 중 개인정보 또는 민감 자료가 포함될 수 있는 항목이 확인되었습니다.

아래 자료는 홈페이지 제작에 사용하지 않겠습니다.

1. 학생 이름이 포함된 수업 사례
2. 학생 얼굴 또는 이름표가 보이는 사진
3. 학부모 연락처나 상담 기록이 포함된 파일

홈페이지 제작에는 학생 이름, 성적, 상담 기록, 학부모 연락처, 시험지 원본이 포함되지 않은 자료만 사용할 수 있습니다.
가능하면 위 항목을 제외한 자료로 다시 보내주세요.
```

판정: `PASS`

이 문안은 고객을 과하게 탓하지 않으면서도 사용 불가 기준을 분명히 한다.

## 7. 결론

가상 고객 회신 접수 리허설 재점검은 통과했다.

현재 문서 흐름은 다음 순서로 이어진다.

```text
발송 문안 템플릿
-> 가상 발송 리허설
-> 회신 접수 기록 양식
-> 가상 접수 판정
-> 최신 기준 재점검
```

아직 실제 고객이 없으므로 실제 발송, 실제 접수, 실제 고객 정보 입력은 하지 않는다.

## 8. 다음 작업

다음 1순위 추천 작업은 실제 고객 없이 진행 가능한 파일럿 시연 URL 준비 상태 점검이다.

점검 대상:

- 고객에게 보여줄 수 있는 샘플 홈페이지 URL이 localhost에 머물러 있는지
- 배포 URL 또는 외부 공유 가능한 미리보기 URL이 필요한지
- 실제 고객 정보 없이도 시연 가능한 샘플 데이터 상태인지
- 내부 제작 화면은 외부에 보여주지 않는다는 경계가 유지되는지
