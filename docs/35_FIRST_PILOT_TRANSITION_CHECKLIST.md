# 시각 GO 이후 첫 파일럿 전환 체크리스트

Status: Draft
Project: homepage
Last Updated: 2026-07-01

---

## 1. 목적

이 문서는 공개 샘플 홈페이지가 묵산 시각 판정에서 `GO`를 받은 뒤, 첫 파일럿 단계로 넘어가기 위해 필요한 준비 항목을 정리한다.

현재 실제 고객은 아직 없다.

따라서 이 문서는 고객에게 무언가를 보내는 실행 문서가 아니라, **시각 승인 이후 무엇을 준비하고 무엇을 보류해야 하는지 나누는 전환 체크리스트**다.

---

## 2. 현재 기준 상태

| 항목 | 상태 |
|---|---|
| 공개 샘플 홈페이지 시각 방향 | `GO` |
| 시각 판정 기록 | `docs/34_LOCAL_DEMO_VISUAL_APPROVAL_RECORD.md` |
| 공식 로컬 frontend 포트 | `5175` |
| 현재 `5175` 상태 | 충돌 시 HOLD |
| 임시 검증 URL `5176` | 이번 세션 검증용 |
| 외부 공유 URL | 아직 없음 |
| 실제 고객 | 아직 없음 |
| 운영 배포 승인 | 아직 아님 |

핵심 결론:

```text
샘플 화면의 시각 방향은 통과했다.
그러나 고객에게 링크를 보낼 준비가 끝난 것은 아니다.
```

---

## 3. 전환 원칙

첫 파일럿 전환은 다음 순서로 나눈다.

```text
시각 GO
-> 외부 공유 URL 방식 결정
-> 고객에게 보낼 자료 요청 패킷 최종 확인
-> 실제 파일럿 고객이 생긴 뒤 저장소 밖에서 고객별 값 입력
-> 고객 자료 요청 발송
-> 회신 접수와 개인정보/민감자료 필터링
-> 내부 제작
-> 고객 확인용 미리보기
-> 게시 전 최종 승인
```

중간 단계를 건너뛰지 않는다.

특히 시각 GO는 외부 공유 URL 승인이나 실제 고객 발송 승인이 아니다.

---

## 4. 즉시 가능한 작업

실제 고객 없이 바로 할 수 있는 작업:

| 작업 | 목적 | 결과물 |
|---|---|---|
| 외부 공유 URL 방식 후보 정리 | 고객이 직접 열 수 있는 URL 준비 방향 결정 | staging/운영 URL 판단 |
| 첫 파일럿 자료 요청 패킷 재확인 | 고객 발송 전 문구와 질문량 확인 | 발송 후보 문안 |
| 파일럿 자료 회신 처리 절차 재점검 | 고객 자료 수신 후 내부 처리 흐름 고정 | 접수 기록 기준 |
| 5175 충돌 해소 또는 임시 URL 정책 유지 판단 | 내부 시연 혼선 방지 | 로컬 시연 절차 안정화 |
| 운영 배포 전 런북 재점검 | 외부 공유 전 서버/HTTPS/DB 기준 확인 | 배포 준비 판단 |

---

## 5. 아직 하면 안 되는 작업

현재 하면 안 되는 작업:

| 작업 | 보류 이유 |
|---|---|
| 실제 고객에게 자료 요청 발송 | 실제 고객이 아직 없다. |
| 실제 고객명/연락처를 저장소 문서에 입력 | 개인정보와 고객별 운영 정보는 저장소 밖에서 다룬다. |
| localhost 또는 127.0.0.1 링크를 고객에게 전달 | 고객 PC에서 열리지 않는다. |
| 외부 공유 URL 없이 고객 확인 요청 | 고객이 직접 확인할 수 없다. |
| `/internal` 화면을 고객에게 공유 | 내부 제작 화면이다. |
| 운영 배포 승인으로 간주 | 시각 GO와 운영 배포는 별도다. |

---

## 6. 외부 공유 URL 판단

첫 파일럿 고객에게 직접 보여주려면 외부 공유 URL이 필요하다.

선택지는 세 가지다.

| 선택지 | 설명 | 현재 판단 |
|---|---|---|
| 화면 공유 기반 시연 | 묵산 또는 내부 담당자가 로컬 화면을 직접 보여준다. | 지금 가능 |
| 임시 staging URL | 고객이 직접 열 수 있는 제한 공개 URL을 만든다. | 다음 판단 필요 |
| 운영 기준 URL | Windows/Caddy/HTTPS 기준으로 실제 운영에 가깝게 배포한다. | 준비 문서는 있으나 실행 전 |

첫 파일럿에서 고객이 직접 확인해야 한다면, 화면 공유만으로는 부족할 수 있다.

그 경우 다음 작업은 staging 또는 운영 URL 방식 결정이 된다.

## 8. 첫 파일럿 GO 조건

실제 첫 파일럿 고객에게 자료 요청을 보내려면 아래 조건이 필요하다.

| 조건 | 필요 판정 |
|---|---|
| 실제 파일럿 고객 존재 | 필요 |
| 고객에게 보낼 브랜드명 | `베틀 시스템` |
| 고객별 정보 입력 위치 | 저장소 밖 |
| 회신 기한 | 확정 필요 |
| 파일 제출 방식 | 확정 필요 |
| 샘플 확인 URL | 외부 공유 가능해야 함 |
| 개인정보/민감자료 차단 문구 | 포함 필요 |
| 내부 전용 문구 제거 | 확인 필요 |

하나라도 빠지면 실제 발송은 HOLD다.

---

## 9. 추천 다음 공정

현재 가장 중요한 분기점은 외부 공유 URL이다.

이유:

- 시각 GO는 이미 받았다.
- 자료 요청 패킷도 가상 리허설을 통과했다.
- 그러나 고객이 직접 열 수 있는 샘플 URL은 아직 없다.
- localhost/127.0.0.1은 고객에게 보낼 수 없다.

따라서 다음 1순위 추천 작업은 **첫 파일럿용 외부 공유 URL 방식 결정**이다.

결정해야 할 것:

- 화면 공유만으로 첫 파일럿 상담을 진행할지
- 제한 공개 staging URL을 먼저 만들지
- Windows/Caddy 운영 배포 리허설을 실제 URL 준비까지 밀고 갈지
