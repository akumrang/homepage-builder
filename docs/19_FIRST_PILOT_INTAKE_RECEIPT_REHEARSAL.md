Status: Draft
Project: homepage
Last Updated: 2026-06-28

---

## 1. 목적

이 문서는 `docs/18_FIRST_PILOT_INTAKE_RECEIPT_RECORD_TEMPLATE.md`를 실제 접수 상황에 적용한다는 가정으로 작성한 가상 판정 예시다.

실제 고객명, 담당자명, 연락처, 원본 사진 링크, 원본 로고 파일 링크, 학생 정보는 기록하지 않는다.

목표는 다음 세 가지다.

- 접수 기록 양식만으로 `MATERIALS_READY` 전환 가능 여부를 판단할 수 있는지 확인
- `MATERIALS_READY`와 `CUSTOMER_PUBLISHED` 전환 조건을 분리할 수 있는지 확인
- 부족 자료 재요청 문구가 실제 운영에 충분한지 확인

---

## 2. 가상 회신 조건

이 예시는 실제 고객 자료가 아니다.

가정:

- 고객은 `docs/17_FIRST_PILOT_CUSTOMER_SEND_COPY.md`를 받고 회신했다.
- 학원 기본 정보, 과목, 대상 학년, 상담 방식, 커리큘럼, 강점 후보는 제출했다.
- 로고 파일은 아직 없지만 텍스트 로고 fallback은 승인했다.
- 대표 사진 후보는 제출했지만 사진 사용 동의 확인은 아직 완료되지 않았다.
- 강사 경력은 증빙 가능한 이력 대신 수업 방식 중심 공개를 선택했다.
- 실제 학생, 보호자, 성적, 결제, 시험지 원본 자료는 제출하지 않았다.

## 6. 권장 자료 접수 체크

| 항목 | 접수 상태 | 메모 |
|---|---|---|
| 대표 사진 또는 교실 사진 | 접수 | 사용 동의 확인 전까지 게시 보류 |
| 로고 | 없음 | 이번 초안은 텍스트 로고 fallback 사용 |
| 텍스트 로고 fallback 승인 | 접수 | 고객 승인 완료 가정 |
| 수업 시간표 1개 이상 | 접수 | 대표 시간표 1개 제출 |
| 초기 공지 1개 이상 | 접수 | 개강 안내 1개 제출 |
| 교통 안내 | 접수 | 지하철/버스 기준 안내 가능 |
| 주차 안내 | 없음 | 생략 가능 |
| 상담 유도 문구 톤 | 접수 | 차분하고 신뢰 중심 |

권장 자료 판정:

```text
권장 자료: SKIP_OK
```

판단:

- 사진 사용 동의는 `CUSTOMER_PUBLISHED` 전환 전 보류 항목이다.
- 주차 안내는 필수 정보가 아니므로 초안 제작을 막지 않는다.
- 로고 파일이 없어도 텍스트 로고 fallback 승인이 있으므로 초안 제작을 막지 않는다.

## 7. Asset 승인 상태

### 7.1 로고

| 항목 | 상태 |
|---|---|
| 로고 파일 제출 | 없음 |
| 로고 출처 | 해당 없음 |
| 로고 사용 동의 | 해당 없음 |
| 텍스트 로고 fallback 승인 | 확인 |

판정:

```text
publication.assets.logo:
- assetId: undefined
- source: MUKSAN_CREATED
- approvedForPublish: false
- textFallbackApproved: true
```

해석:

- 로고 파일은 없지만 텍스트 로고 fallback은 고객이 승인했다.
- 현재 코드 기준으로 실제 로고 asset이 없을 때는 `textFallbackApproved: true`가 게시 게이트의 로고 조건을 충족한다.

### 7.2 대표 사진

| 항목 | 상태 |
|---|---|
| 대표 사진 제출 | 있음 |
| 사용 가능한 사진 | 확인 필요 |
| 사진 사용 동의 | 미확인 |
| 개인정보 노출 사진 제외 | 완료 |
| 대체 이미지 필요 | 판단 대기 |

판정:

```text
publication.assets.hero:
- assetId: [운영 문서에서만 기록]
- source: CUSTOMER_PROVIDED
- approvedForPublish: false
```

해석:

- 사진 후보는 있으므로 초안 제작 자료로는 충분하다.
- 사진 사용 동의가 미확인이라 고객 게시 전환은 보류한다.
- 동의 확인이 지연되면 묵산 승인 대체 이미지를 별도로 준비한다.

## 9. 제작 상태 판정

| 상태 | 가능 여부 | 조건 |
|---|---|---|
| `WAITING_FOR_MATERIALS` 유지 | 아니오 | 필수 자료가 충족됨 |
| `MATERIALS_READY` 전환 | 예 | 필수 자료와 공개 가능성 확인 완료 |
| `DRAFT_CREATED` 진행 | 예 | 묵산 초안 작성 가능 |
| `CUSTOMER_REVIEW` 진행 | 예 | 내부 검수 후 고객 확인 가능 |
| `CUSTOMER_PUBLISHED` 전환 | 아니오 | 사진 사용 동의와 고객 최종 게시 승인 필요 |

최종 판정:

```text
접수 판정: MATERIALS_READY 가능 / 게시 전 보류
```

상태 전환:

```text
WAITING_FOR_MATERIALS → MATERIALS_READY
```

아직 전환하지 않는 상태:

```text
CUSTOMER_PUBLISHED
```

## 11. 양식 검증 결과

`docs/18_FIRST_PILOT_INTAKE_RECEIPT_RECORD_TEMPLATE.md`는 현재 가상 상황을 기록하기에 충분하다.

확인된 점:

- 필수 자료 충족 여부를 `READY`로 판정할 수 있다.
- 권장 자료 부족을 `SKIP_OK`로 분리할 수 있다.
- 로고 파일 없음과 텍스트 로고 fallback 승인을 분리할 수 있다.
- 사진 후보 있음과 사진 사용 동의 미확인을 분리할 수 있다.
- `MATERIALS_READY` 전환과 `CUSTOMER_PUBLISHED` 보류를 같은 기록 안에서 분리할 수 있다.

보완 후보:

- 실제 운영 문서에서는 원본 파일 저장 위치와 내부 asset ID를 별도 비공개 필드로 관리해야 한다.
- 향후 내부 화면을 만들 경우, `초안 제작 가능`과 `게시 가능`을 같은 체크로 묶지 않아야 한다.

## 12. 다음 작업

첫 파일럿 자료 수집 문서 흐름은 고객 발송, 회신 접수, 가상 판정까지 이어졌다.

다음 1순위는 내부 제작 화면 접근 제어 설계 문서를 작성하는 것이다.

이유:

- 현재 내부 화면은 MVP용 접근 키 기반이다.
- 첫 파일럿 고객 자료를 다루기 전에는 내부 접근 범위와 운영자 책임 경계를 더 명확히 해야 한다.
- 고객 포털, 고객 계정, 자유형 편집기는 계속 보류하되, 묵산 내부 화면 보호 기준은 선행되어야 한다.
