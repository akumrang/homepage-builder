# 공개 페이지 게시 모드 정책

Status: Draft
Project: homepage
Last Updated: 2026-06-27

---

## 1. 목적

이 문서는 `trust-basic-v1` 공개 홈페이지를 샘플 화면에서 실제 고객 게시 화면으로 전환할 때 지켜야 할 기준을 정의한다.

핵심 목적은 세 가지다.

- 샘플 고지 문구와 고객 게시 문구를 분리한다.
- 실제 학원 사진, 로고, 주소, 연락처를 게시하기 전 검수 기준을 둔다.
- 고객 게시용 footer와 샘플 footer를 분리한다.

이 정책은 고객용 자유형 편집기나 무제한 맞춤 제작을 의미하지 않는다. 묵산 표준 템플릿과 검수 절차 안에서 공개 상태를 관리하기 위한 기준이다.

---

## 2. `productionStatus`와 게시 모드의 분리

`productionStatus`는 내부 제작 흐름 상태다.

예:

```text
REQUESTED
WAITING_FOR_MATERIALS
MATERIALS_READY
DRAFT_CREATED
INTERNAL_REVIEW
CUSTOMER_REVIEW
APPROVED
PUBLISHED
```

게시 모드는 공개 페이지가 어떤 맥락으로 렌더링되는지를 나타내는 별도 축이다.

예:

```text
SAMPLE
CUSTOMER_PREVIEW
CUSTOMER_PUBLISHED
```

따라서 `productionStatus=PUBLISHED`만으로 실제 고객 게시가 완료되었다고 판단하지 않는다.

현재 MVP의 `sample-korean-academy`는 샘플 홈페이지이므로 내부 제작 상태가 `PUBLISHED`여도 게시 모드 관점에서는 `SAMPLE`이다.

---

## 3. 게시 모드 정의

| 모드 | 목적 | 공개 범위 | 핵심 기준 |
|---|---|---|---|
| `SAMPLE` | 묵산 샘플, 내부 QA, 향후 샘플 갤러리 후보 | 내부 또는 제한 공개 | 샘플임을 명확히 고지하고 실제 개인정보를 쓰지 않는다 |
| `CUSTOMER_PREVIEW` | 고객 최종 확인용 초안 | 고객과 묵산 내부 제한 공유 | 실제 고객 자료를 쓰되, 게시 전 확인용임을 표시한다 |
| `CUSTOMER_PUBLISHED` | 실제 고객 학원 공개 홈페이지 | 일반 공개 | 샘플 고지를 제거하고 승인된 실제 정보와 asset만 쓴다 |

---

## 4. 샘플 모드 기준

`SAMPLE` 모드에서는 다음을 허용한다.

- 샘플 학원명
- 샘플 주소
- 샘플 전화번호
- 샘플 강사진
- 샘플 공지
- 생성 이미지 또는 템플릿 기본 이미지
- 실제 개인정보 미사용 고지

`SAMPLE` 모드에서는 다음을 금지한다.

- 실제 학원처럼 오해될 수 있는 실적, 후기, 경력
- 실제 학생, 보호자, 강사 개인정보
- 실제 결제 링크처럼 보이는 동작
- 실제 고객 학원과 혼동되는 브랜드 표현

샘플 footer 기준:

```text
<학원명>
샘플 홈페이지 · 실제 개인정보 없음
상담 문의
```

샘플 footer에도 `/internal` 같은 내부 제작 화면 링크는 노출하지 않는다.

## 5. 고객 미리보기 모드 기준

`CUSTOMER_PREVIEW` 모드는 실제 고객 자료를 반영하지만 아직 일반 공개 전인 상태다.

목적:

- 고객이 자기 학원의 이름, 문구, 사진, 로고, 주소, 연락처를 최종 확인한다.
- 묵산이 허위 실적, 허위 후기, 미검증 경력, 개인정보 노출을 검수한다.
- 게시 전 수정 요청을 받는다.

표시 기준:

- 일반 공개 SEO 대상으로 삼지 않는다.
- 고객 확인용이라는 맥락을 내부 또는 제한 공유 화면에서 식별 가능하게 한다.
- 샘플 고지 문구는 제거할 수 있지만, 일반 공개처럼 오해되지 않게 운영자가 관리한다.

`CUSTOMER_PREVIEW`에서 확인할 항목:

- 학원명과 로고 사용 승인
- 대표 문구와 상담 문구 승인
- hero 사진 또는 대체 이미지 승인
- 주소, 연락처, 운영 시간 승인
- 강사진 공개명과 경력 사실 확인
- 커리큘럼 표현 승인
- 공지 공개 여부 승인
- 개인정보 미포함 확인

## 6. 고객 게시 모드 기준

`CUSTOMER_PUBLISHED`는 실제 고객 학원의 공개 홈페이지다.

게시 전 필수 조건:

- 고객이 최종 게시를 승인했다.
- 샘플 학원명, 샘플 주소, 샘플 전화번호가 남아 있지 않다.
- 샘플 고지 문구가 남아 있지 않다.
- 실제 학원명, 주소, 연락처, 상담 가능 시간이 확인됐다.
- 실제 로고 또는 텍스트 로고 사용 방식이 확정됐다.
- hero 사진이 고객 제공 asset이거나 묵산 승인 대체 이미지다.
- 사진, 로고, 강사 프로필 사용 동의가 확인됐다.
- 허위 실적, 허위 후기, 미검증 경력이 없다.
- footer가 고객 게시용 문구로 전환됐다.
- `/internal` 또는 내부 제작 화면 진입 링크가 공개 footer에 없다.

고객 게시 footer 기준:

```text
<학원명>
<주소 또는 운영 고지>
상담 문의
```

사업자 정보, 개인정보 처리방침, 학원 등록 정보 등 법적 표시가 필요해지면 별도 필드와 별도 정책으로 추가한다.

## 7. 사진과 로고 적용 기준

### 7.1 로고

고객 로고가 있으면 다음을 확인한다.

- 고객이 홈페이지 사용을 승인했다.
- 모바일 header에서 잘리지 않는다.
- 배경 대비가 충분하다.
- 파일이 너무 작거나 흐리지 않다.

고객 로고가 없으면 텍스트 로고로 시작할 수 있다. 이 경우 고객 게시 전 텍스트 표기와 지점명 포함 여부를 확인한다.

### 7.2 Hero 사진

hero 사진 우선순위:

1. 고객이 제공하고 사용을 승인한 실제 학원 사진
2. 고객이 승인한 묵산 촬영 또는 제작 asset
3. 실제 학원 공간처럼 오해를 만들지 않는 템플릿 기본 이미지

금지:

- 다른 학원 사진 사용
- 무단 stock 이미지 사용
- 실제 학원 시설처럼 보이지만 고객 학원과 무관한 이미지 사용
- 학생 얼굴이 식별되는 사진을 동의 없이 사용

### 7.3 강사 사진

강사 사진은 본인 또는 고객의 공개 동의가 있는 경우에만 사용한다.

동의가 없으면 현재 MVP처럼 이니셜 마크, 공개명, 수업 방식 중심 소개로 대체한다.

## 8. 문구 전환 기준

샘플 모드에서 허용되는 문구:

- 샘플 홈페이지
- 실제 개인정보 없음
- 샘플 공개 프로필
- 샘플 주소
- 샘플 안내

고객 게시 모드에서 제거해야 하는 문구:

- 샘플
- 실제 개인정보 없음 또는 실제 개인정보 미사용 고지
- 샘플 공개 프로필
- 확인되지 않은 경력이나 실적은 표시하지 않습니다
- MVP에서는 실제 결제와 외부 시스템 연동 없이

단, 허위 실적 방지 원칙 자체는 유지한다. 고객 게시 화면에서는 샘플 방어 문구를 노출하기보다, 미검증 내용을 애초에 게시하지 않는 방식으로 처리한다.

## 9. 공개 전 GO / NO-GO 체크

| 항목 | GO 기준 | NO-GO 기준 |
|---|---|---|
| 게시 모드 | `CUSTOMER_PUBLISHED`로 판정 가능 | 샘플인지 고객 게시인지 불명확 |
| 학원명 | 고객 승인 명칭 | `샘플` 또는 임시명 잔존 |
| 연락처 | 고객 승인 공개 번호 | 더미 번호 또는 미확인 번호 |
| 주소 | 고객 승인 공개 주소 | 샘플 주소 또는 미확인 주소 |
| 로고 | 고객 승인 로고 또는 승인된 텍스트 로고 | 무단 로고, 깨진 이미지 |
| hero | 승인된 사진 또는 승인 대체 이미지 | 실제 시설 오해 가능 이미지 |
| footer | 고객 게시용 문구 | 샘플 고지 또는 내부 링크 잔존 |
| 강사진 | 공개명과 내용 승인 | 미검증 경력, 무단 사진 |
| 공지 | 공개 여부 확인 | 내부 공지 또는 샘플 공지 잔존 |
| 개인정보 | 공개 가능 정보만 있음 | 학생/보호자 개인정보 노출 |

하나라도 `NO-GO`이면 일반 공개 게시를 중단한다.

## 10. 현재 MVP와의 관계

현재 구현은 게시 모드 초안 필드를 갖고 있다.

현재 상태:

- `sample-korean-academy`는 샘플 모드로 취급한다.
- footer는 `publication` 필드 기준으로 샘플/고객 미리보기/고객 게시 문구를 분리해 렌더링한다.
- `CUSTOMER_PUBLISHED` 검증 fixture로 샘플 고지, 샘플 asset 경로, MVP 방어 문구 잔존 여부를 자동 확인한다.
- 고객 게시 모드에서는 entry band의 MVP 방어 문구를 고객 게시용 안내 문구로 전환한다.
- `publication.assets.logo`와 `publication.assets.hero`로 로고/hero asset의 출처와 게시 승인 상태를 분리한다.
- 고객 게시 모드에서는 샘플 출처 hero, 미승인 hero, 승인되지 않은 logo asset, 승인되지 않은 텍스트 로고 fallback을 `NO-GO`로 판정한다.
- 자료 수집 양식의 logo/hero 입력은 `mapIntakeAssetsToPublicationAssets` helper를 통해 `publication.assets` metadata로 변환한다.
- 내부 화면의 콘텐츠 탭에서 logo, hero, 게시 게이트 상태를 읽기 전용 승인 준비도 패널로 표시한다.
- 실제 asset 파일 저장소나 asset 업로드 UI는 아직 없다. 현재는 게시 전 내부 검증용 metadata만 둔다.

현재 초안:

```ts
type PublicationMode = "SAMPLE" | "CUSTOMER_PREVIEW" | "CUSTOMER_PUBLISHED";

type PublicationAssetSource =
  | "SAMPLE"
  | "CUSTOMER_PROVIDED"
  | "MUKSAN_CREATED"
  | "MUKSAN_APPROVED_REPLACEMENT";

interface AcademyPublicationPolicy {
  mode: PublicationMode;
  sampleDisclosureVisible: boolean;
  customerApprovedForPublish: boolean;
  assets: {
    logo: {
      assetId?: string;
      source: PublicationAssetSource;
      approvedForPublish: boolean;
      textFallbackApproved: boolean;
    };
    hero: {
      assetId?: string;
      source: PublicationAssetSource;
      approvedForPublish: boolean;
    };
  };
  footerNote?: string;
}
```

이 구현은 고객용 자유 편집기를 뜻하지 않는다. 묵산 내부 제작자가 게시 가능 여부를 판단하고, 공개 렌더링 문구를 안전하게 바꾸기 위한 최소 장치다.

## 11. 다음 작업 후보

이 정책 이후의 구현 후보는 다음이다.

```text
첫 파일럿 자료 요청 패킷 실고객 발송 전 최종 검토
→ 질문량, 발송 문안 톤, 파일 제출 안내, 공개 동의 문구를 묵산이 최종 확인한다
→ 고객 계정과 고객용 자유 편집기는 계속 보류
```

다만 실제 고객 파일럿 전에는 먼저 고객 자료 수집 양식과 실제 asset 준비 상태를 확인해야 한다.
