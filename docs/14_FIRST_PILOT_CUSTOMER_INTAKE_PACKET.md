## 1. 목적

이 문서는 첫 고객 파일럿에서 묵산이 고객 학원에게 보낼 홈페이지 제작 자료 요청 패킷이다.

목표는 고객에게 홈페이지 제작툴을 제공하는 것이 아니라, 묵산 내부 제작자가 표준 템플릿 안에서 완성형 홈페이지를 제작할 수 있을 만큼의 공개 정보, 디자인 선호, asset 사용 승인, 검수 동의를 안정적으로 받는 것이다.

첫 파일럿에서는 이 문서 양식을 사용하고, 별도 고객 포털이나 자료 수집 API는 만들지 않는다.

## 2. 고객 발송 안내문

아래 문안은 고객에게 그대로 보낼 수 있다.

```text
안녕하세요. 묵산입니다.

학원 홈페이지 초안 제작을 위해 아래 자료를 요청드립니다.

이번 홈페이지는 고객님이 직접 편집하는 제작툴이 아니라, 묵산이 표준 템플릿 안에서 학원별 정보를 정리하여 완성형 홈페이지로 제작하는 방식입니다.

보내주신 자료는 공개 가능 여부를 확인한 뒤 사용하며, 학생 이름, 학부모 연락처 목록, 성적표, 결제 정보, 시험지 원본 등 개인정보나 민감 자료는 받지 않습니다.

가능한 범위에서 답변해 주시면 묵산이 문구와 구성을 정리해 초안을 제작하고, 게시 전 최종 확인을 요청드리겠습니다.
```

## 4. 파일 제출 안내

### 4.1 로고

가능하면 다음 중 하나를 제출한다.

- `png`
- `jpg`
- `svg`
- 원본 디자인 파일이 있다면 별도 첨부

파일명 예시:

```text
academy-logo-01.png
academy-logo-white.png
academy-logo-original.svg
```

로고가 없으면 텍스트 로고로 시작할 수 있다. 이 경우 학원명 표기를 홈페이지 상단 로고처럼 사용해도 되는지 확인한다.

### 4.2 대표 사진

가능하면 다음 사진 중 1-3장을 제출한다.

- 교실
- 입구
- 상담 공간
- 복도 또는 안내 데스크
- 실제 수업 분위기를 보여줄 수 있는 공간 사진

파일명 예시:

```text
academy-hero-classroom-01.jpg
academy-entrance-01.jpg
academy-counseling-room-01.jpg
```

사진 주의사항:

- 학생 얼굴, 학생 이름표, 성적표, 학부모 연락처가 보이면 사용하지 않는다.
- 사람이 찍힌 사진은 초상권 동의가 있어야 한다.
- 실제 학원 공간이 아닌 이미지가 실제 시설처럼 오해되면 사용하지 않는다.
- 사진이 부족하면 묵산 승인 대체 이미지 또는 템플릿 기본 이미지로 시작한다.

### 4.3 보내지 말아야 할 자료

다음 자료는 보내지 않는다.

- 학생 이름, 학교, 성적, 상담 기록
- 학부모 연락처 목록
- 결제 카드 정보, 계좌 비밀번호, PG 인증 정보
- 시험지 원본, 해설 원본, 수업자료 원본 전체
- academy DB dump
- exam_system2 DB dump
- 확인되지 않은 합격 실적
- 허위 후기
- 경쟁 학원 사이트를 그대로 복제하라는 요청
- CSS, HTML, JavaScript 직접 수정 요청

## 5. 확인과 동의 문구

고객에게 다음 항목을 확인받는다.

```text
[ ] 제출한 학원명, 주소, 연락처, 운영 시간, 수업 정보는 홈페이지에 공개 가능한 정보입니다.

[ ] 제출 자료에 학생 이름, 학부모 연락처 목록, 성적, 결제 정보, 시험지 원본 등 개인정보나 민감 자료가 포함되어 있지 않습니다.

[ ] 제출한 로고와 사진은 홈페이지 제작과 게시에 사용할 수 있습니다.

[ ] 사람 얼굴이 포함된 사진은 초상권과 사용 동의를 확인했습니다.

[ ] 강사 경력이나 이력은 사실 확인이 가능한 내용만 제출했습니다.

[ ] 홈페이지 초안은 게시 전 최종 확인 절차를 거친 뒤 공개된다는 점에 동의합니다.
```

필수 확인:

- 공개 가능 정보 확인
- 개인정보 미포함 확인
- 사진 사용 동의
- 강사 경력 사실 확인
- 게시 전 최종 확인 절차 동의

## 7. 부족 자료 재요청 문안

```text
자료 확인 감사합니다.

홈페이지 초안 제작을 위해 아래 항목만 추가로 확인 부탁드립니다.

1. [부족 항목]
2. [부족 항목]
3. [부족 항목]

해당 항목이 준비되면 묵산이 초안 제작을 진행하고, 게시 전 최종 확인 화면을 공유드리겠습니다.
```

## 8. 시스템 반영 기준

묵산 내부 제작자는 고객 답변을 다음 기준으로 반영한다.

| 고객 자료 | 내부 반영 |
|---|---|
| 학원명 | `AcademySite.name` |
| 대표 문구 초안 | `AcademySite.tagline` 또는 묵산 작성 문구 |
| 수업 요약 | `AcademySite.summary`, `introduction`, `curriculum` |
| 과목과 학년 | `subjects`, `targetGrades` |
| 강사 정보 | `teachers` |
| 상담 정보 | `consultation` |
| 주소와 연락처 | `location` |
| 공지 | `notices` |
| 로고 자료 | `publication.assets.logo` |
| 대표 사진 | `publication.assets.hero`, `heroImage` |

자료가 충분하지 않으면 상태를 `WAITING_FOR_MATERIALS`로 유지한다. 필수 자료와 공개 가능성 확인이 충분하면 `MATERIALS_READY`로 전환한다.

고객 게시 전에는 `CUSTOMER_PUBLISHED` 전환 조건을 별도로 확인한다.

## 9. 첫 파일럿 후 기록할 것

첫 파일럿이 끝나면 다음을 기록한다.

- 고객이 답변하기 어려워한 항목
- 반복적으로 누락된 항목
- 로고와 사진 자료 준비 상태
- 고객 수정 요청 횟수
- 묵산 내부 제작 소요 시간
- 문서 양식으로 충분했는지
- 내부 입력 화면이 필요한지
- 자료 수집 API로 승격할 명확한 이유가 생겼는지

이 기록을 기준으로 자료 수집 API와 내부 입력 화면 구현 여부를 다시 판단한다.
