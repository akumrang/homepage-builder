## 1. 목적

이 문서는 첫 파일럿 고객에게 실제로 보낼 자료 요청 사본의 템플릿이다.

실제 고객명, 담당자명, 연락처, 이메일, 회신 기한 같은 실고객 정보는 저장소에 커밋하지 않는다. 발송 직전에 묵산이 로컬 문서, 메일 본문, 메신저 본문에서만 채운다.

이 사본은 `docs/14_FIRST_PILOT_CUSTOMER_INTAKE_PACKET.md`와 `docs/16_FIRST_PILOT_INTAKE_PACKET_FINAL_REVIEW.md`를 바탕으로 만든다.

---

## 2. 발송 전 채울 값

아래 값은 실제 발송 시에만 채운다.

| 항목 | 값 |
|---|---|
| 학원명 | `[학원명]` |
| 담당자 호칭 | `[원장님/실장님/담당자님]` |
| 회신 기한 | `[YYYY-MM-DD]` |
| 파일 제출 방식 | `[이메일 첨부 / 드라이브 링크 / 메신저 첨부]` |
| 묵산 문의 연락처 | `[묵산 문의 연락처]` |
| 샘플 홈페이지 링크 | `[샘플 홈페이지 링크]` |

주의:

- 실제 고객 연락처, 이메일, 개인명은 이 저장소에 기록하지 않는다.
- 발송용 완성본은 고객별 외부 문서나 메일 초안에서 관리한다.

## 3. 고객 발송 본문

```text
안녕하세요, [학원명] [담당자 호칭].
묵산 홈페이지 초안 제작을 위해 자료 요청 양식을 보내드립니다.

이번 홈페이지는 고객님이 직접 편집하는 제작툴이 아니라, 묵산이 표준 템플릿 안에서 학원별 정보를 정리하여 완성형 홈페이지로 제작하는 방식입니다.

모든 항목을 완벽히 채우지 않으셔도 됩니다.
가능한 항목부터 보내주시면 묵산이 부족한 부분을 확인해 다시 요청드리겠습니다.

로고나 사진이 없더라도 우선 제작을 시작할 수 있습니다.
다만 공개 가능한 정보, 사진 사용 동의, 개인정보 미포함 여부는 게시 전 반드시 확인합니다.

회신 기한: [YYYY-MM-DD]
파일 제출 방식: [이메일 첨부 / 드라이브 링크 / 메신저 첨부]
샘플 확인 링크: [샘플 홈페이지 링크]

확인 중 궁금한 점이 있으면 [묵산 문의 연락처]로 편하게 말씀 주세요.
```

## 5. 파일 제출 안내

### 5.1 로고

가능하면 `png`, `jpg`, `svg` 파일 중 하나를 제출한다.

```text
[ ] 로고 파일이 없습니다. 우선 학원명을 텍스트 로고처럼 사용해도 됩니다.
[ ] 로고 파일을 제출합니다. 홈페이지 제작과 게시에 사용할 수 있습니다.
[ ] 로고 파일은 나중에 제출하겠습니다.
```

### 5.2 대표 사진

가능하면 다음 사진 중 1-3장을 제출한다.

- 교실
- 입구
- 상담 공간
- 복도 또는 안내 데스크
- 실제 수업 분위기를 보여줄 수 있는 공간 사진

사진 제출 NO-GO:

```text
학생 얼굴이 보이는 사진은 사용하지 않습니다.
학생 이름표, 성적표, 상담 기록, 학부모 연락처가 보이는 사진은 사용하지 않습니다.
사람이 찍힌 사진은 초상권과 게시 동의가 확인된 경우에만 사용합니다.
```

## 6. 보내지 말아야 할 자료

다음 자료는 보내지 않는다.

- 학생 이름, 학교, 성적, 상담 기록
- 학부모 연락처 목록
- 결제 카드 정보, 계좌 비밀번호, PG 인증 정보
- 시험지 원본, 해설 원본, 수업자료 원본 전체
- academy DB dump
- exam_system2 DB dump
- 확인되지 않은 합격 실적
- 허위 후기
- 경쟁 학원 사이트를 그대로 복제하라는 요청
- CSS, HTML, JavaScript 직접 수정 요청

## 7. 확인과 동의

```text
[ ] 제출한 학원명, 주소, 연락처, 운영 시간, 수업 정보는 홈페이지에 공개 가능한 정보입니다.

[ ] 제출 자료에 학생 이름, 학부모 연락처 목록, 성적, 결제 정보, 시험지 원본 등 개인정보나 민감 자료가 포함되어 있지 않습니다.

[ ] 제출한 로고와 사진은 홈페이지 제작과 게시에 사용할 수 있습니다.

[ ] 로고가 없으면 학원명을 텍스트 로고처럼 사용하는 것에 동의합니다.

[ ] 사람 얼굴이 포함된 사진은 초상권과 사용 동의를 확인했습니다.

[ ] 강사 경력이나 이력은 사실 확인이 가능한 내용만 제출했습니다. 확인이 어려운 경력은 공개하지 않고 수업 방식 중심으로 소개해도 됩니다.

[ ] 홈페이지 초안은 게시 전 최종 확인 절차를 거친 뒤 공개된다는 점에 동의합니다.
```

## 9. 다음 작업

실제 발송 후에는 회신 자료를 기준으로 자료 접수 기록을 남긴다.

기록할 것:

- 회신일
- 필수 자료 충족 여부
- 부족 자료
- NO-GO 자료
- `MATERIALS_READY` 전환 가능 여부
- `CUSTOMER_PUBLISHED` 전환 전 남은 확인 항목

실제 고객 개인정보나 민감 자료는 저장소에 기록하지 않는다.

자료 회신 후 내부 접수 기록 양식은 `docs/18_FIRST_PILOT_INTAKE_RECEIPT_RECORD_TEMPLATE.md`를 사용한다.
