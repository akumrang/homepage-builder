# 첫 파일럿 자료 회신 접수 기록 양식

Status: Draft
Project: homepage
Last Updated: 2026-06-28

---

## 1. 목적

이 문서는 첫 파일럿 고객이 자료 요청 패킷에 회신한 뒤, 묵산 내부 제작자가 접수 상태를 판정하기 위한 기록 양식이다.

이 문서에는 실제 고객 개인정보, 연락처, 학생 정보, 원본 사진, 원본 파일 링크를 기록하지 않는다. 실제 고객별 접수 기록은 별도 운영 문서에서 관리한다.

이 저장소에는 접수 기록의 구조와 판정 기준만 둔다.

---

## 2. 사용 시점

다음 순서에서 사용한다.

1. `docs/17_FIRST_PILOT_CUSTOMER_SEND_COPY.md`를 고객에게 발송한다.
2. 고객이 자료를 회신한다.
3. 묵산 내부 제작자가 회신 자료에서 개인정보와 민감 자료를 먼저 걸러낸다.
4. 이 양식으로 필수 자료, 권장 자료, NO-GO, asset 승인 상태를 판정한다.
5. `WAITING_FOR_MATERIALS`, `MATERIALS_READY`, `CUSTOMER_REVIEW`, `CUSTOMER_PUBLISHED` 가능 여부를 기록한다.

## 3. 접수 기본 정보

실제 값은 운영 문서에만 입력한다.

| 항목 | 기록 |
|---|---|
| 접수 ID | `[운영 문서에서 생성]` |
| 고객 학원 식별자 | `[실명 대신 내부 식별자 권장]` |
| 회신일 | `[YYYY-MM-DD]` |
| 회신 채널 | `[이메일 / 드라이브 / 메신저 / 기타]` |
| 자료 검수자 | `[묵산 내부 담당자]` |
| 현재 제작 상태 | `WAITING_FOR_MATERIALS` |
| 다음 목표 상태 | `MATERIALS_READY` |

## 4. 개인정보와 민감 자료 1차 차단

이 단계에서 하나라도 `있음`이면 해당 자료는 홈페이지 제작에 사용하지 않는다.

| 항목 | 없음/있음 | 조치 |
|---|---|---|
| 학생 이름 |  | 있으면 삭제 또는 재제출 요청 |
| 학교명과 개인 식별 정보 조합 |  | 있으면 삭제 또는 재제출 요청 |
| 성적, 상담 기록, 학습 이력 |  | 있으면 삭제 또는 재제출 요청 |
| 학부모 연락처 목록 |  | 있으면 삭제 또는 재제출 요청 |
| 결제 정보 |  | 있으면 즉시 폐기 |
| 시험지 원본 또는 해설 원본 |  | 있으면 사용하지 않음 |
| 사진 속 학생 얼굴 |  | 있으면 해당 사진 사용 금지 |
| 사진 속 이름표, 성적표, 연락처 |  | 있으면 해당 사진 사용 금지 |

판정:

```text
개인정보/민감 자료 1차 차단: PASS / HOLD
```

## 7. Asset 승인 상태

### 7.1 로고

| 항목 | 상태 |
|---|---|
| 로고 파일 제출 | 있음 / 없음 / 나중에 제출 |
| 로고 출처 | 고객 제공 / 묵산 제작 / 묵산 승인 대체 / 해당 없음 |
| 로고 사용 동의 | 확인 / 미확인 / 해당 없음 |
| 텍스트 로고 fallback 승인 | 확인 / 미확인 / 해당 없음 |

판정:

```text
publication.assets.logo:
- assetId: [운영 문서에서만 기록]
- source: CUSTOMER_PROVIDED / MUKSAN_CREATED / MUKSAN_APPROVED_REPLACEMENT
- approvedForPublish: true / false
- textFallbackApproved: true / false
```

### 7.2 대표 사진

| 항목 | 상태 |
|---|---|
| 대표 사진 제출 | 있음 / 없음 / 나중에 제출 |
| 사용 가능한 사진 | 있음 / 없음 / 확인 필요 |
| 사진 사용 동의 | 확인 / 미확인 |
| 개인정보 노출 사진 제외 | 완료 / 필요 |
| 대체 이미지 필요 | 필요 / 불필요 |

판정:

```text
publication.assets.hero:
- assetId: [운영 문서에서만 기록]
- source: CUSTOMER_PROVIDED / MUKSAN_CREATED / MUKSAN_APPROVED_REPLACEMENT
- approvedForPublish: true / false
```

## 9. 제작 상태 판정

| 상태 | 가능 여부 | 조건 |
|---|---|---|
| `WAITING_FOR_MATERIALS` 유지 | 예 / 아니오 | 필수 자료 부족 또는 NO-GO 미해결 |
| `MATERIALS_READY` 전환 | 예 / 아니오 | 필수 자료와 공개 가능성 확인 완료 |
| `DRAFT_CREATED` 진행 | 예 / 아니오 | 묵산 초안 작성 가능 |
| `CUSTOMER_REVIEW` 진행 | 예 / 아니오 | 내부 검수 후 고객 확인 가능 |
| `CUSTOMER_PUBLISHED` 전환 | 예 / 아니오 | 고객 승인, 샘플 문구 제거, asset 승인, NO-GO 없음 |

최종 판정:

```text
접수 판정: MATERIALS_READY 가능 / 추가 자료 요청 / 게시 전 보류
```

## 11. 내부 기록 원칙

저장소에 기록하지 않는다.

- 실제 고객명과 담당자 실명
- 고객 연락처와 이메일
- 원본 사진 파일 링크
- 원본 로고 파일 링크
- 학생 또는 학부모 개인정보
- 상담 기록
- 성적, 학교, 학습 이력

저장소에 기록해도 되는 것은 다음뿐이다.

- 양식 구조
- 판정 기준
- placeholder
- 실제 고객을 특정할 수 없는 운영 절차
