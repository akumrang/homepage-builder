## 2. 최종 판정

판정: `READY FOR MUKSAN FILL-IN`

고객 발송 자료의 구조와 문구는 첫 파일럿 기준으로 사용 가능하다.

다만 실제 발송 전에는 묵산이 저장소 밖에서 다음 값을 채워야 한다.

- 고객 학원명
- 담당자 호칭
- 회신 기한
- 파일 제출 방식
- 샘플 홈페이지 링크
- 베틀 시스템 문의 연락처

실제 고객명, 개인 연락처, 이메일, 고객별 제출 링크는 저장소에 커밋하지 않는다.

`docs/17_FIRST_PILOT_CUSTOMER_SEND_COPY.md`에 다음을 추가했다.

## 5. 실제 발송 직전 묵산이 할 일

발송 직전에는 저장소 파일을 직접 고치지 않는다.

묵산은 로컬 메일 초안, 메신저 본문, 또는 고객별 외부 문서에서만 다음 값을 채운다.

| 항목 | 작성 기준 |
|---|---|
| 고객 학원명 | 실제 학원명 |
| 담당자 호칭 | 원장님, 실장님, 담당자님 등 |
| 회신 기한 | 첫 파일럿은 3-5영업일 권장 |
| 파일 제출 방식 | 이메일 첨부, 드라이브 링크, 메신저 첨부 중 하나 |
| 샘플 홈페이지 링크 | 배포 URL 또는 내부 확인용 URL |
| 베틀 시스템 문의 연락처 | 실제 응답 가능한 연락 방식 |

고객에게 보낼 때는 `docs/17_FIRST_PILOT_CUSTOMER_SEND_COPY.md`의 고객 발송 범위만 복사한다.

## 6. 발송하면 안 되는 경우

다음 중 하나라도 해당하면 실제 발송을 보류한다.

- 실제 고객명이나 연락처를 저장소 문서에 직접 적어야 하는 상황
- 회신 기한 또는 파일 제출 방식이 아직 정해지지 않은 상황
- 샘플 홈페이지 링크가 아직 고객에게 보여줄 상태가 아닌 상황
- 문의를 받을 실제 연락 방식이 정해지지 않은 상황
- 고객에게 내부 발송 체크나 내부 공정 문구까지 함께 보낼 위험이 있는 상황

## 7. 다음 작업

다음 1순위 추천 작업은 묵산의 실제 발송 승인이다.

묵산이 확인할 것:

- 첫 파일럿 대상 고객이 정해졌는가
- 회신 기한을 정할 수 있는가
- 파일 제출 방식을 정할 수 있는가
- 샘플 홈페이지 링크를 고객에게 보여줘도 되는가
- 베틀 시스템 문의 연락처를 확정할 수 있는가

이 다섯 가지가 준비되면 실제 발송으로 넘어간다.
