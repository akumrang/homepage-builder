# 파일럿 시연 URL 준비 상태 점검

Status: Draft
Project: homepage
Last Updated: 2026-06-30

---

## 1. 목적

이 문서는 실제 고객이 아직 없는 현재 단계에서, 파일럿 시연에 사용할 수 있는 URL의 준비 상태를 판단한다.

핵심 질문은 다음이다.

- 샘플 홈페이지를 내부 시연으로 보여줄 수 있는가?
- 외부에 그대로 전달할 수 있는 URL이 이미 있는가?
- 실제 고객 정보 없이 시연 가능한가?
- 내부 제작 화면과 공개 홈페이지의 노출 경계가 유지되는가?

---

## 2. 결론

현재 상태의 결론:

| 항목 | 판정 | 설명 |
|---|---|---|
| 로컬 내부 시연 URL | 가능 | `http://127.0.0.1:5175/h/sample-korean-academy`로 샘플 홈페이지를 확인할 수 있다. |
| 실제 고객 정보 없는 시연 | 가능 | `sample-korean-academy` 샘플 데이터와 테스트 문의만 사용하면 된다. |
| 외부 공유용 URL | 아직 없음 | `127.0.0.1`과 `localhost`는 이 PC에서만 열리는 주소이므로 고객에게 보낼 수 없다. |
| 운영 배포용 URL | 미완료 | Windows/Caddy 운영 계획과 리허설 문서는 있으나 실제 도메인, HTTPS, 서버 배포는 아직 수행하지 않았다. |
| 내부 제작 화면 외부 노출 | 금지 | `/internal`은 베틀 시스템 내부 제작자 확인용이며 외부 시연 대상이 아니다. |

따라서 지금 할 수 있는 시연은 **개발 PC 또는 화면 공유 기반 내부 시연**이다.

고객에게 링크를 보내서 직접 열게 하는 단계는 아직 아니다.

---

## 3. 현재 사용할 수 있는 URL

로컬 개발 서버 기준:

| 용도 | URL | 사용 가능 범위 |
|---|---|---|
| 공개 샘플 홈페이지 | `http://127.0.0.1:5175/h/sample-korean-academy` | 내부 시연 |
| 내부 제작 화면 | `http://127.0.0.1:5175/internal` | 내부 제작자 확인 |
| backend health | `http://localhost:4200/api/health` | 개발/운영 점검 |
| backend readiness | `http://localhost:4200/api/ready` | 개발/운영 점검 |

위 URL은 로컬 실행 상태에서만 유효하다.

외부 공유용 URL로 오해하면 안 된다.

---

## 4. 실제 고객 없이 가능한 시연 범위

가능:

- 공개 샘플 홈페이지 첫 화면 확인
- 소개, 강사진, 커리큘럼, 공지, 오시는 길 확인
- 상담 문의 폼 입력
- 개인정보 수집 동의 누락 시 제출 차단 확인
- 테스트 상담 문의 제출
- 내부 제작 화면에서 테스트 문의 접수 상태 확인
- 로컬 개발 DB를 정리한 뒤 깨끗한 상태로 시연

조건:

- 보호자명, 연락처, 문의 내용은 모두 가짜 테스트 정보만 사용한다.
- `/internal` 화면은 고객에게 보여주지 않는다.
- 시연 전 `docs/23_PILOT_DEMO_LOCAL_DB_CLEANUP_RUNBOOK.md`에 따라 로컬 DB를 정리한다.

---

## 5. 아직 준비되지 않은 것

외부 공유 또는 실제 고객 확인에 필요한 항목:

| 항목 | 상태 |
|---|---|
| staging 또는 preview 도메인 | 미정 |
| HTTPS 인증서 | 미발급 |
| 실제 서버 provision | 미수행 |
| Caddy runtime 설정 적용 | 문서/템플릿 단계 |
| Windows Service 실제 등록 | 문서/템플릿 단계 |
| 외부 공유 전 내부 접근 차단 재점검 | 필요 |
| 공개 샘플 URL의 문구 최종 확인 | 필요 |

운영 배포 기준은 다음 문서에 이미 분리되어 있다.

- `docs/04_HOSTING_REVERSE_PROXY_PLAN.md`
- `docs/07_MVP_PRODUCTION_ENVIRONMENT_DECISION.md`
- `docs/08_WINDOWS_CADDY_SERVICE_TEMPLATES.md`
- `docs/09_WINDOWS_OPERATION_REHEARSAL_CHECKLIST.md`
- `docs/10_WINDOWS_DEPLOYMENT_QUICKSTART.md`

이 문서는 배포를 실행하지 않고, 현재 URL 준비 상태만 판정한다.

## 6. 시연 방식 판단

### 6.1 지금 즉시 가능한 방식

개발 PC에서 frontend/backend를 실행하고, 화면 공유 또는 현장 브라우저로 보여준다.

```text
개발 PC 브라우저 또는 화면 공유
-> http://127.0.0.1:5175/h/sample-korean-academy
-> 테스트 상담 문의 제출
-> /internal에서 접수 확인
```

이 방식은 실제 고객 정보가 필요 없고, 외부 URL도 필요 없다.

### 6.2 아직 보류해야 하는 방식

고객에게 직접 링크를 보내는 방식은 아직 보류한다.

이유:

- 현재 링크가 `localhost` 또는 `127.0.0.1`이다.
- 고객 PC에서는 열리지 않는다.
- 실제 공유 전 HTTPS, 도메인, 내부 화면 차단, 테스트 데이터 정리 확인이 필요하다.

## 7. 판정

파일럿 시연 URL 준비 상태 점검 결과:

```text
로컬 내부 시연: READY
외부 공유 URL: NOT READY
실제 고객 정보 필요 여부: NOT NEEDED
내부 제작 화면 외부 노출: BLOCKED BY POLICY
```

현재 단계에서는 실제 고객을 상정한 발송이나 승인 요청을 하지 않는다.

필요한 것은 베틀 시스템 내부에서 샘플 홈페이지가 시연 화면으로 충분한지 브라우저로 다시 확인하는 것이다.

## 8. 다음 작업

다음 1순위 추천 작업은 로컬 시연 URL 브라우저 확인이다.

확인할 내용:

- `http://127.0.0.1:5175/h/sample-korean-academy`가 현재 샘플 시연 화면으로 충분한지
- 모바일/데스크톱에서 첫 화면과 상담 CTA가 자연스러운지
- 실제 고객 정보 없이 테스트 문의를 제출해도 흐름이 매끄러운지
- `/internal`은 내부 제작자 확인용으로만 유지되는지

외부 링크 발송 단계로 넘어가려면 그 다음에 staging 또는 운영 URL 방식을 별도 결정한다.
