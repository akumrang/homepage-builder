# 첫 파일럿 자료 요청 패킷 최종 검토

Status: Draft  
Project: homepage  
Last Updated: 2026-06-30

---

## 1. 목적

이 문서는 `docs/14_FIRST_PILOT_CUSTOMER_INTAKE_PACKET.md`를 실제 고객에게 보내기 전 베틀 시스템이 확인해야 할 최종 검토 기록이다.

검토 대상은 고객에게 보낼 자료 요청 패킷의 질문량, 문안 톤, 파일 제출 안내, 공개 동의 문구, 개인정보 차단 기준이다.

이 문서는 고객용 문서가 아니다. 베틀 시스템 내부 발송 승인 기준이다.

---

## 2. 검토 결론

판정: `GO WITH LIGHT EDIT`

첫 파일럿 고객에게 발송 가능하다. 다만 실제 발송 전에는 고객명, 발송 채널, 회신 기한, 파일 제출 방식만 베틀 시스템이 채워야 한다.

구현 확대는 필요하지 않다.

- 고객 포털: 보류
- 자료 수집 API: 보류
- 내부 intake 입력 화면: 보류
- 파일 업로드 시스템: 보류
- 문서 양식 운영: 진행

---

## 3. 발송 전 베틀 시스템이 채울 항목

| 항목 | 상태 | 메모 |
|---|---|---|
| 고객 학원명 | 발송 전 입력 | 메일 또는 메시지 첫 문장에 반영 |
| 담당자 호칭 | 발송 전 입력 | 원장님, 실장님 등 |
| 회신 기한 | 발송 전 입력 | 첫 파일럿은 3-5영업일 권장 |
| 파일 제출 방식 | 발송 전 입력 | 이메일 첨부, 드라이브 링크, 메신저 첨부 중 하나 |
| 문의 연락처 | 발송 전 입력 | 베틀 시스템 담당자 연락 방식 |
| 샘플 홈페이지 링크 | 발송 전 입력 | `/h/sample-korean-academy` 또는 배포 URL |

발송 문안에 위 항목이 빠지면 고객이 어디로, 언제까지, 어떤 방식으로 자료를 보내야 하는지 헷갈릴 수 있다.

## 4. 질문량 검토

질문량은 첫 파일럿 기준으로 허용 가능하다.

다만 고객 부담을 줄이기 위해 발송 시 다음 안내를 함께 붙인다.

```text
모든 항목을 완벽히 채우지 않으셔도 됩니다.
가능한 항목부터 보내주시면 베틀 시스템이 부족한 부분을 확인해 다시 요청드리겠습니다.
```

판정:

| 항목 | 판정 |
|---|---|
| 기본 정보 | 적정 |
| 수업 정보 | 적정 |
| 강사진 정보 | 약간 민감하지만 필요 |
| 상담 정보 | 적정 |
| 디자인 선호 | 적정 |
| 초기 공지 | 선택 항목으로 적정 |
| 확인과 동의 | 필수 |

## 5. 문안 톤 검토

현재 문안은 고객에게 과도한 자동화나 제작툴 판매처럼 보이지 않는다.

좋은 점:

- 베틀 시스템이 완성형 홈페이지를 제작한다는 점이 명확하다.
- 고객이 직접 편집하는 제작툴이 아니라는 점이 명확하다.
- 개인정보와 민감 자료를 받지 않는다는 기준이 명확하다.
- 게시 전 최종 확인 절차가 명시되어 있다.

주의할 점:

- `자료 요청` 문서가 길어 보일 수 있다.
- 발송 메시지 본문에서는 “가능한 만큼만 먼저 보내달라”는 문장을 추가한다.
- 고객에게 책임을 떠넘기는 표현처럼 보이지 않도록, 베틀 시스템이 문구를 정리한다는 점을 함께 설명한다.

## 6. 파일 제출 안내 검토

로고와 사진 제출 안내는 첫 파일럿 기준으로 충분하다.

GO 기준:

- 로고가 없어도 텍스트 로고로 시작 가능하다는 안내가 있다.
- 대표 사진이 없어도 베틀 시스템 승인 대체 이미지 또는 템플릿 기본 이미지로 시작할 수 있다.
- 사진에 학생 얼굴, 이름표, 성적표, 상담 기록, 연락처가 있으면 사용하지 않는다는 NO-GO가 있다.
- 강사 사진 제출과 게시 동의가 별도라는 점이 명확하다.

발송 전 추가할 운영 안내:

```text
사진은 원본 그대로 보내주셔도 됩니다.
베틀 시스템이 사용 가능 여부를 확인하고, 문제가 있는 사진은 게시하지 않겠습니다.
```

## 7. 공개 동의와 개인정보 차단 검토

현재 동의 문구는 첫 파일럿 기준으로 필요 최소 수준을 충족한다.

필수 확인:

- 공개 가능 정보 확인
- 개인정보 미포함 확인
- 로고와 사진 사용 동의
- 텍스트 로고 fallback 승인
- 초상권과 게시 동의
- 강사 경력 사실 확인
- 게시 전 최종 확인 절차 동의

NO-GO:

- 학생 이름, 학교, 성적, 상담 기록
- 학부모 연락처 목록
- 결제 정보
- 시험지 원본
- 미검증 합격 실적
- 허위 후기
- 외부 사이트 복제 요청
- CSS, HTML, JavaScript 직접 수정 요청

판정:

`GO`. 다만 실제 고객이 체크박스를 모두 회신하지 않으면 `CUSTOMER_PUBLISHED` 전환 전 재확인을 해야 한다.

## 8. 고객에게 보낼 짧은 발송 본문

아래 문안을 자료 요청 패킷 앞에 붙여 발송한다.

```text
안녕하세요, [학원명] [담당자 호칭].
베틀 시스템 홈페이지 초안 제작을 위해 자료 요청 양식을 보내드립니다.

모든 항목을 완벽히 채우지 않으셔도 됩니다.
가능한 항목부터 보내주시면 베틀 시스템이 부족한 부분을 확인해 다시 요청드리겠습니다.

로고나 사진이 없더라도 우선 제작을 시작할 수 있습니다.
다만 공개 가능한 정보, 사진 사용 동의, 개인정보 미포함 여부는 게시 전 반드시 확인합니다.

회신 기한: [회신 기한]
파일 제출 방식: [이메일 첨부 / 드라이브 링크 / 메신저 첨부]
샘플 확인 링크: [샘플 홈페이지 링크]

확인 중 궁금한 점이 있으면 [문의 연락처]로 편하게 말씀 주세요.
```

## 9. 내부 처리 순서

자료를 받은 뒤 베틀 시스템은 다음 순서로 처리한다.

1. 개인정보 또는 민감 자료 포함 여부 확인
2. 필수 자료 누락 여부 확인
3. 로고, 사진, 강사 경력, 공지 공개 여부 검수
4. 부족 자료 재요청
5. `MATERIALS_READY` 전환 가능 여부 판단
6. `AcademySite` 초안 작성
7. `content:validate`와 내부 콘텐츠 탭으로 점검
8. 고객 확인용 미리보기 공유
9. 고객 승인 후 `CUSTOMER_PUBLISHED` 전환 판단
