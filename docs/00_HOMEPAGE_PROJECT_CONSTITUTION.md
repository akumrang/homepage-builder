# 묵산 홈페이지 프로젝트 헌법 v1

## 1. 프로젝트 정체성

묵산 홈페이지 프로젝트는 홈페이지 제작툴을 판매하는 사업이 아니다.

묵산 SaaS를 구독하는 학원에게 학원당 1개의 홈페이지를 제작·제공하고, 이를 상담, 공지, 수강료 결제, 학원관리, 시험지 생성관리로 연결하는 통합 접점이다.

## 2. 전체 SaaS 안에서의 위치

묵산 SaaS는 세 축으로 구성된다.

1. 시험지 생성관리 프로그램
2. 학원관리 프로그램
3. 홈페이지 제작·관리 시스템

홈페이지는 독립 상품이 아니라 묵산 SaaS의 외부 얼굴이다.

## 3. 핵심 원칙

- 고객에게 홈페이지 제작툴을 제공하지 않는다.
- 묵산 내부 제작 시스템으로 학원당 1개의 홈페이지를 제작한다.
- 누적 3천~1만 개 학원 홈페이지를 감당할 수 있어야 한다.
- 1인 또는 극소수 운영이 가능하도록 반복 업무를 자동화한다.
- 싸구려 제품은 만들지 않는다.
- AI는 초안 생성 보조로 사용하되, 최종 결과물은 묵산 기준과 검수를 통과해야 한다.

## 4. 최상위 판단 기준

이 설계는 묵산 1인이 높은 품질을 유지하면서 장기적으로 수천 개 학원 홈페이지를 관리할 수 있게 하는가?
