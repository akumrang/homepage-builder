## 1. 프로젝트 정체성

묵산 홈페이지 프로젝트는 홈페이지 제작툴을 판매하는 사업이 아니다.

묵산 SaaS를 구독하는 학원에게 학원당 1개의 고품질 홈페이지를 제작·제공하고, 이를 상담, 공지, 수강료 결제, 학원관리, 시험지 생성관리로 연결하는 통합 접점이다.

정확한 제품 정체성은 고객별 완성 홈페이지 제공 플랫폼이다. 묵산은 여러 유형의 완성형 샘플과 표준 템플릿을 준비하고, 고객 학원은 그중 선호 방향을 선택한다. 묵산 내부 제작자는 학원 자료를 받아 공통 시스템 위에서 각 학원에 맞는 홈페이지를 제작, 검수, 게시한다.

홈페이지는 묵산 SaaS의 부속 장식물이 아니라, 학부모와 학생이 묵산 고객 학원을 처음 만나는 외부 얼굴이다.

## 2. 전체 SaaS 안에서의 위치

묵산 SaaS는 세 축으로 구성된다.

| 축 | 역할 |
|---|---|
| 프로젝트1 시험지 생성관리 | 수업자료, 시험지, 해설, PDF 생성 |
| 프로젝트2 학원관리 | 학생, 보호자, 반, 출결, 수납, 문자, 상담 관리 |
| 프로젝트3 홈페이지 | 학원 홍보, 상담 유입, 학부모 접점, 관리자 진입 |

홈페이지 프로젝트는 독립된 웹에이전시 상품이 아니라, 프로젝트1과 프로젝트2를 외부 사용자에게 연결하는 문이다.

## 3. 규모 원칙

이 시스템은 1만 개 홈페이지를 한 번에 찍어내는 시스템이 아니다.

다만 장기적으로 누적 최소 3천 개, 최대 1만 개 학원 홈페이지 제작과 관리를 감당할 수 있어야 한다.

따라서 다음 구조는 원칙적으로 피한다.

- 고객별 임의 코드
- 무제한 맞춤 디자인
- 수작업 중심 유지보수
- 전화지원 의존 구조
- 고객별 별도 코드베이스 생성

---

## 4. 1인 운영 원칙

묵산은 가능하면 1인 기업 또는 극소수 운영 구조를 지향한다.

따라서 반복 업무는 사람이 아니라 시스템이 처리해야 한다.

자동화 대상은 다음과 같다.

- 학원 정보 수집
- AI 초안 생성
- 템플릿 반영
- 미리보기 생성
- 고객 확인
- 게시
- 공지 연결
- 상담 문의 연결
- 결제 링크 연결
- 장애 감시
- 백업
- 기본 고객 안내

묵산은 반복 작업자가 아니라 최종 판단자이자 품질 승인자여야 한다.

## 5. 품질 원칙

묵산은 싸구려 제품을 만들지 않는다.

홈페이지는 덤으로 끼워주는 허접한 페이지처럼 보여서는 안 된다.

학부모가 보았을 때 다음 인상이 들어야 한다.

- 신뢰감
- 실제 운영감
- 전문성
- 상담 욕구
- 안정감

자동화는 품질을 낮추기 위한 수단이 아니라, 높은 기준을 반복 가능하게 만드는 수단이다.

## 6. 사용자 구분

홈페이지는 다음 사용자를 기준으로 설계한다.

| 사용자 | 핵심 목적 |
|---|---|
| 예비 학부모 | 검색 후 첫인상, 신뢰 판단, 상담 신청 |
| 예비 학생 | 학원 분위기, 과목, 커리큘럼 확인 |
| 재원 학부모 | 공지, 결제, 상담, 학원 소식 확인 |
| 원장·강사 | 관리자 시스템 진입 |
| 묵산 운영자 | 제작, 검수, 게시, 관리 |

입학 전에는 홈페이지가 중요하다.

재원 후에는 모바일 웹 또는 향후 학부모 앱으로 발전할 수 있는 접점이 중요하다.

## 7. 제작 방식

고객에게 자유형 홈페이지 제작툴을 제공하지 않는다.

고객은 묵산이 미리 준비한 완성형 홈페이지 샘플을 보고 선호 방향을 선택한다. 고객이 직접 레이아웃을 자유롭게 편집하거나 컴포넌트를 무제한 조합하는 구조가 아니다.

묵산 내부 제작 시스템이 고객 선택값과 학원 정보를 받아 표준 템플릿 기반으로 홈페이지를 제작한다.

기본 흐름은 다음과 같다.

```text
완성형 샘플 확인
→ 고객의 선호 방향 선택
→ 학원 정보 수집
→ AI 초안 생성
→ 묵산 표준 템플릿 반영
→ 묵산 검수
→ 고객 확인
→ 게시
→ 제한된 유지관리
```

고객이 선택할 수 있는 범위는 다음과 같다.

- 홈페이지 유형
- 색감 방향
- 첫 화면 강조점
- 강사진 중심, 커리큘럼 중심, 상담 중심 같은 콘텐츠 우선순위
- 공지 노출 방식
- 상담 CTA 문구

고객이 직접 할 수 없는 것은 다음과 같다.

- 자유형 드래그앤드롭 편집
- 무제한 레이아웃 재배치
- 디자인 시스템을 깨는 색상, 폰트, 컴포넌트 변경
- 고객별 임의 코드 추가
- academy 또는 exam_system2 데이터 구조에 영향 주는 설정

## 8. AI 사용 원칙

AI는 제작자가 아니라 보조자다.

AI가 할 수 있는 일:

학원 소개 문구 초안
첫 화면 문구
강점 정리
커리큘럼 설명
강사진 소개 초안
상담 유도 문구
FAQ 초안
SEO 제목과 설명 초안
템플릿 추천

AI가 해서는 안 되는 일:

합격 실적 창작
학생 또는 학부모 후기 창작
강사 경력 창작
결제 또는 개인정보 처리 흐름 임의 생성
고객별 임의 코드 홈페이지 납품
검수 없이 바로 게시

AI 결과는 반드시 묵산 검수와 고객 확인을 거친다.

## 11. 데이터 경계

홈페이지는 데이터를 직접 소유하는 거대한 제3 시스템이 되어서는 안 된다.

데이터 소유권은 다음을 원칙으로 한다.

데이터	소유 시스템
학생, 보호자, 반, 수납, 문자, 상담 기록	academy
시험지, 문제, 해설, PDF, 수업자료	exam_system2
공개 콘텐츠, 템플릿, 게시 상태, 도메인, 상담 진입 정보	homepage

상담 문의와 결제 진입은 장기적으로 academy와 연결한다.

시험지 생성관리 결과는 필요한 경우 샘플, 소개, 관리자 진입 형태로만 연결한다.

## 12. 최종 정의

묵산 홈페이지 프로젝트는 다음과 같이 정의한다.

구독 학원마다 개별 완성 홈페이지를 제공하되, 이를 공통 템플릿·학원별 콘텐츠 데이터·묵산 내부 제작 시스템으로 반복 생산하고 운영하며, 상담·공지·결제·학원관리·시험지 생성관리로 연결하는 묵산 SaaS의 외부 접점 플랫폼.

이 헌법의 최상위 기준은 다음이다.

1인 또는 극소수 운영자가 높은 품질을 유지하면서 누적 3천~1만 개 학원 홈페이지를 감당할 수 있는가?
