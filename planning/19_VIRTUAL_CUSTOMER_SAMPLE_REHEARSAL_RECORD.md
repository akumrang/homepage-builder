## 4. 상담 흐름 리허설

### Step 1. 제작 방식 안내

상담 문구:

```text
베틀 시스템 홈페이지는 고객이 직접 만드는 자유형 빌더가 아니라, 베틀 시스템이 준비한 표준 방향과 템플릿 안에서 내부 제작자가 완성하는 방식입니다.
오늘은 최종 디자인을 확정하는 자리가 아니라, 학원에 맞는 제작 방향을 고르는 단계입니다.
```

판정:

```text
PASS
```

근거:

- 자유형 빌더가 아님을 먼저 설명한다.
- 방향 선택과 최종 디자인 확정을 구분한다.

보완 필요:

- “표준 방향과 템플릿”이라는 표현은 내부자에게는 명확하지만 고객에게는 다소 딱딱할 수 있다.
- 고객 공유용 문구에서는 “베틀 시스템이 준비한 제작 방향”처럼 더 쉬운 표현이 낫다.

### Step 2. 고객 권리와 제한 안내

상담 흐름:

- 고객은 분위기, 대표 문구, 색상 선호, 사진 방향, 강조 장점을 제공할 수 있다.
- 묵산은 레이아웃 무제한 변경, 섹션 자유 조립, 코드 수정, 타 사이트 복제는 제공하지 않는다.

판정:

```text
PASS
```

근거:

- 고객의 선택권과 제한을 함께 설명했다.
- “제한”이 아니라 “베틀 시스템이 완성하는 제작 방식”으로 설명하면 방어적으로 들리지 않는다.

보완 필요:

- 상담 공유용 자료에서는 고객 권리 목록을 먼저 보여주고, 제한은 “제작 안정성을 위해 제공하지 않는 것”으로 이어붙이는 순서가 좋다.

### Step 3. 샘플 방향 3종 설명

상담 문구:

```text
신뢰형 기본은 가장 안정적인 기본 방향입니다.
내신/입시 관리형은 시험 대비 체계가 분명한 학원에 맞지만, 실적 표현은 확인 자료가 있어야 합니다.
소수정예 관리형은 작은 반과 꾸준한 피드백이 실제 강점인 학원에 맞습니다.
```

판정:

```text
PASS
```

근거:

- 세 방향의 차이가 짧게 전달된다.
- 내신/입시 관리형의 실적 자료 조건을 즉시 설명한다.

보완 필요:

- 고객에게 처음 보여줄 때는 세 방향의 “좋고 나쁨”보다 “어떤 학원에 맞는지”가 먼저 와야 한다.
- “신뢰형 기본은 가장 안정적”이라는 표현은 첫 선택지를 낮춰 보이게 만들 수 있으므로 “처음 제작에 가장 부담이 적은 방향”처럼 바꾸는 것이 좋다.

### Step 4. 실제 샘플 확인

확인 경로:

```text
/h/sample-korean-academy
```

확인 순서:

1. 첫 화면
2. 상담 CTA
3. 핵심 강점
4. 소개
5. 강사진
6. 커리큘럼
7. 공지
8. 오시는 길과 상담 신청
9. footer 샘플 고지

판정:

```text
PASS
```

근거:

- 실제 구현된 `trust-basic-v1`을 기준으로 고객이 완성도와 정보 구조를 확인할 수 있다.

보완 필요:

- 현재 구성안의 순서는 첫 화면 이후 핵심 강점으로 넘어가지만, 상담에서는 CTA를 먼저 짚어주는 편이 좋다.
- footer 샘플 고지는 마지막이 아니라 시작과 끝에서 모두 설명하는 것이 안전하다.

### Step 5. 고객 질문 대응

질문과 답변 판정:

| 질문 | 답변 방향 | 판정 |
|---|---|---|
| 이걸 고르면 그대로 만들어지나요? | 방향 참고안이며 자료와 검수에 따라 조정 | PASS |
| 샘플끼리 섞을 수 있나요? | 일부 강조점은 반영 가능, 자유 조립은 불가 | PASS |
| 우리 학원 색으로 바꿀 수 있나요? | 선호는 가능, 디자인 토큰 안에서 조정 | PASS |
| 다른 사이트랑 비슷하게 만들 수 있나요? | 분위기 참고는 가능, 복제는 불가 | PASS |
| 성적 상승 사례나 후기를 넣을 수 있나요? | 확인 자료 있을 때만 가능 | PASS |
| 선택하면 바로 게시되나요? | 제작, 검수, 고객 확인 후 게시 | PASS |

보완 필요:

- “일부 강조점은 반영 가능”은 좋지만 “섹션을 섞는다”는 표현을 상담자가 반복하지 않는 편이 안전하다.
- “분위기 참고”는 허용하되, 외부 사이트 URL을 받는 경우 복제 기대가 생기지 않도록 별도 문구가 필요하다.

### Step 6. 자료 준비 체크

가상 고객 자료 판정:

| 항목 | 상태 | 판정 | 다음 조치 |
|---|---|---|---|
| 학원명 | 있음 | GO | 공식 표기 확인 |
| 로고 | 없음 | HOLD | 텍스트 로고 승인 요청 |
| 대표 문구 | 없음 | HOLD | 1문장 초안 요청 |
| 대표 색상 선호 | 없음 | 선택 | 선호 색상 또는 피하고 싶은 색상 확인 |
| 사진 | 일부 있음 | HOLD | 학생 식별 여부 검수 |
| 강사진 소개 | 미정 | HOLD | 이름 공개 여부와 역할 중심 소개 요청 |
| 커리큘럼 | 초안 있음 | HOLD | 학년별 과정과 상담 가능 반 정리 |
| 공지 | 없음 | 선택 | 개강/상담 공지 1개 요청 가능 |
| 주소/전화/운영 시간 | 미정 | HOLD | 공개 가능 정보 요청 |
| 실적/후기 | 근거 없음 | NO-GO | 사용하지 않음 |

판정:

```text
HOLD
```

근거:

- 제작 방향 상담은 가능하다.
- 실제 제작 착수에는 로고/텍스트 로고, 대표 문구, 강사진, 주소/전화/운영 시간이 부족하다.
- 실적/후기는 근거가 없으므로 제외해야 한다.

## 5. 최종 판정

```text
HOLD
```

이유:

- 고객 기대 관리는 대체로 통제 가능하다.
- 직접 편집 기능에 대한 질문은 표준 답변으로 방어 가능하다.
- 다만 제작 착수에는 필수 공개 정보가 부족하다.
- 실적/후기 근거가 없어 해당 콘텐츠는 제외해야 한다.

추천 방향:

```text
1순위: 신뢰형 기본
2순위: 소수정예 관리형
보류: 내신/입시 관리형
```

판정 근거:

- 현재 고객은 작은 반과 과제 점검을 강조하지만, 기본 자료가 부족하다.
- 첫 제작은 `trust-basic-v1`을 기준으로 안전하게 시작하고, 실제 관리 루틴이 충분히 정리되면 소수정예 톤을 일부 반영하는 것이 적절하다.
- 내신/입시 관리형은 실적/성과 자료 없이 강조하면 과장 위험이 크다.

## 6. 발견한 보완점

문구 보완:

- “표준 방향과 템플릿”보다 “베틀 시스템이 준비한 제작 방향”이 고객에게 더 자연스럽다.
- “가장 안정적인 기본 방향”보다 “처음 제작에 부담이 적고, 수업 흐름을 차분히 보여주는 방향”이 낫다.
- “일부 강조점은 반영 가능”은 유지하되, 샘플 섹션 조립 가능성으로 들리지 않게 해야 한다.

자료 순서 보완:

1. 제작 방식 안내
2. 샘플은 실제 개인정보 없는 참고 자료라는 고지
3. 고객이 선택할 수 있는 것
4. 제공하지 않는 것
5. 세 방향 한눈 비교
6. 실제 샘플 `trust-basic-v1` 확인
7. 후보 방향은 브리프 단계임을 안내
8. 자료 준비 체크
9. 내부 GO/HOLD/NO-GO 판정

추가 필요 문구:

```text
외부 사이트는 참고 분위기를 파악하기 위한 자료로만 볼 수 있습니다.
묵산은 타 사이트를 복제하지 않고, 표준 템플릿과 검수 기준 안에서 새로 제작합니다.
```

## 7. 리허설 결론

현재 정적 샘플 자료는 내부 상담 리허설에 사용할 수 있다.

다만 고객 공유용 문구는 조금 더 부드럽게 조정할 필요가 있다. 특히 “표준 템플릿”, “제한”, “기본 방향” 같은 표현은 내부 문서에는 적합하지만 고객에게는 다소 건조하게 들릴 수 있다.
