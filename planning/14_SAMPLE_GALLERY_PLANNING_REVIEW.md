# 샘플 갤러리 기획 재검토

Status: Draft  
Project: homepage  
Last Updated: 2026-06-28

---

## 1. 목적

이 문서는 완성형 샘플 갤러리를 즉시 구현할지 판단하기 위한 기획 재검토 기록이다.

결론부터 말하면 샘플 갤러리는 제품 전략상 유지한다. 다만 지금 단계의 결론은 구현 착수가 아니라, 고객 디자인 권리 2단계인 `방향 선택형`을 어떤 범위 안에서 제공할지 정리하는 것이다.

샘플 갤러리는 고객용 자유형 홈페이지 제작툴이 아니다. 고객이 고르는 것은 제작 방향이며, 최종 홈페이지는 묵산의 표준 템플릿, 디자인 토큰, 내부 검수, 고객 확인 절차 안에서 완성된다.

---

## 2. 현재 판단

| 항목 | 판단 |
|---|---|
| 샘플 갤러리의 전략 가치 | 유지 |
| 현재 즉시 구현 여부 | 보류 |
| 현재 필요한 작업 | 기획 재검토와 선행 조건 정리 |
| 고객 권리 성격 | 방향 선택권 |
| 고객에게 주지 않을 권리 | 자유 레이아웃 편집권, 코드 수정권, 무한 맞춤 요구권 |

샘플 갤러리는 단순 영업 자료가 아니라 고객이 자기 학원의 얼굴에 대해 제한적 디자인 권리를 행사하는 첫 관문이다.

다만 이 관문은 고객이 직접 홈페이지를 만드는 도구가 아니다. 묵산이 준비한 여러 완성형 방향 중 선호를 확인하고, 그 선호를 묵산 내부 제작자가 제작 판단에 반영하는 절차다.

---

## 3. 고객에게 보여줄 의미

고객에게 전달할 핵심 문구는 다음 방향이어야 한다.

```text
아래 샘플은 홈페이지 제작 방향을 고르기 위한 참고 화면입니다.
선택하신 방향은 최종 디자인 확정이 아니며, 학원 자료 확인과 묵산 검수 후 제작됩니다.
레이아웃 자유 편집, 코드 수정, 타 사이트 복제 방식의 맞춤 제작은 제공하지 않습니다.
```

고객이 이해해야 할 것:

- 샘플은 완성 홈페이지의 분위기와 정보 구조를 보여주는 참고안이다.
- 고객은 학원명, 로고, 대표 색상 선호, 사진, 문구 톤, 강조 장점을 제공한다.
- 고객은 최종 게시 전 확인과 수정 요청을 할 수 있다.
- 고객의 선택은 묵산 내부 제작의 입력값이지 자동 게시 명령이 아니다.

고객이 오해하면 안 되는 것:

- 샘플의 모든 레이아웃을 마음대로 섞을 수 있다는 뜻이 아니다.
- 고객이 직접 섹션을 조립하는 빌더가 아니다.
- 특정 외부 사이트와 똑같이 만들어준다는 뜻이 아니다.
- 선택 즉시 홈페이지가 자동 생성되어 게시된다는 뜻이 아니다.

## 4. 초기 노출 방식 판단

첫 단계의 샘플 갤러리는 일반 공개 서비스 화면보다 상담/제작 보조 자료에 가깝게 시작하는 편이 안전하다.

권장 단계:

1. 묵산 내부용 샘플 방향 목록 문서
2. 상담 중 고객에게 보여줄 수 있는 반공개 샘플 화면
3. 고객 선호 방향을 자료 수집 기록에 남기는 절차
4. 최소 2~3개 방향의 품질이 확보된 뒤 고객용 갤러리 화면 검토

즉시 하지 않을 것:

- 공개 `/gallery` 또는 `/samples` 라우트 구현
- 고객 선택 제출 API 구현
- 고객 계정 또는 고객 포털 구현
- 샘플 선택값을 자동으로 홈페이지 데이터에 반영
- 다중 템플릿 전체 렌더링 체계 구현

## 5. 최소 샘플 방향 후보

샘플 갤러리를 실제 화면으로 만들기 전에는 최소 2~3개의 방향 브리프가 필요하다.

초기 후보:

| 후보 | 성격 | 적합한 학원 | 주의점 |
|---|---|---|---|
| `trust-basic-v1` | 신뢰형 기본 | 국어, 보습, 중등 중심 학원 | 현재 샘플의 기준 방향. 과장 없이 안정감 유지 |
| `exam-focus-v1` | 내신/입시 관리형 | 시험 대비, 고등부, 관리형 학원 | 허위 합격 실적, 점수 상승 보장 문구 금지 |
| `small-group-v1` | 소수정예형 | 소규모 정원, 밀착 관리 학원 | 프리미엄 과장, 검증되지 않은 강사 경력 금지 |

이 후보명은 코드 구현 약속이 아니다. 다음 단계에서 각 방향의 실제 섹션 구성, 문구 톤, 필요한 사진, 운영 난이도를 먼저 정리해야 한다.

## 6. 샘플 카드에 필요한 정보

향후 갤러리를 구현한다면 각 샘플 카드에는 다음 정보가 필요하다.

| 항목 | 목적 |
|---|---|
| 샘플명 | 고객이 분위기를 쉽게 식별 |
| 방향 태그 | 신뢰형, 입시형, 소수정예형 등 |
| 적합한 학원 | 어떤 학원에 맞는지 안내 |
| 맞지 않는 경우 | 기대 불일치 방지 |
| 대표 섹션 | 메인, 소개, 강사진, 커리큘럼, 공지, 오시는 길 등 |
| 필요한 자료 | 로고, 대표 사진, 강사 소개, 커리큘럼, 공지 |
| 가능한 조정 | 색상 선호, 대표 문구, 사진, 상담 문구 톤 |
| 불가능한 조정 | 자유 레이아웃, 코드 수정, 타 사이트 복제 |
| 검수 기준 | 허위 실적, 후기, 경력, 개인정보 금지 |

## 7. 선택값 처리 원칙

첫 구현 전까지 선택값은 자동 처리하지 않는다.

권장 처리:

```text
고객 선호 방향
→ 자료 수집 기록 또는 상담 메모에 저장
→ 묵산 내부 제작자가 검토
→ 샘플 방향과 고객 자료의 적합성 판단
→ 제작 초안 생성
→ 내부 검수
→ 고객 확인
→ 게시
```

선택값이 곧바로 템플릿, 색상, 섹션 노출을 바꾸는 기능은 아직 만들지 않는다. 이 기능은 제한 편집형 또는 운영자 편집형 단계에서 다시 판단한다.

## 8. 구현 착수 전 GO / NO-GO

GO 조건:

- 최소 2~3개 샘플 방향 브리프가 문서로 정리되어 있다.
- 각 방향에 대해 허용 조정과 금지 조정이 명확하다.
- 고객 기대 관리 문구가 준비되어 있다.
- 샘플 모드와 고객 게시 모드의 분리 기준이 유지된다.
- 실제 개인정보, 허위 실적, 허위 후기, 허위 강사 경력이 없다.
- 묵산 1인 또는 극소수 운영으로 처리 가능한 제작 흐름이다.

NO-GO 조건:

- `trust-basic-v1` 하나뿐인데 갤러리를 공개한다.
- 고객이 선택하면 바로 게시되는 것처럼 보인다.
- 자유형 빌더처럼 보이는 UI가 들어간다.
- 샘플마다 별도 코드베이스가 필요해진다.
- 고객이 외부 사이트 복제를 요구하기 쉬운 문구가 들어간다.
- 샘플 선택 제출 기능을 먼저 만들고 운영 검수 절차를 나중에 붙인다.

## 9. 이번 재검토 결론

샘플 갤러리는 폐기하지 않는다.

하지만 현재 단계의 1순위는 갤러리 구현이 아니라 샘플 방향 브리프 작성이다. 샘플 방향 브리프 없이는 갤러리를 만들어도 고객에게 보여줄 기준이 부족하고, 고객용 자유형 빌더처럼 오해될 위험이 커진다.

현재 권장 순서:

1. 샘플 방향 3종 브리프 작성
2. 샘플 카드 문안과 고객 기대 관리 문구 초안 작성
3. 묵산 내부 검토용 정적 샘플 자료 구성
4. 고객에게 보여줄 기대 관리 문구 확정
5. 그 이후 샘플 갤러리 화면 구현 여부 판단

샘플 방향 3종 브리프는 `planning/15_SAMPLE_DIRECTION_BRIEFS.md`에 둔다.
