# 홈페이지 시스템 경계와 통합 원칙

Status: Draft  
Project: homepage  
Last Updated: 2026-06-24

---

## 1. 목적

이 문서는 묵산 홈페이지 프로젝트가 학원관리 프로그램과 시험지 생성관리 프로그램 사이에서 어떤 역할을 맡는지 정의한다.

홈페이지는 세 번째 축이지만, 모든 데이터를 직접 소유하는 거대한 독립 시스템이 되어서는 안 된다.

---

## 2. 세 프로젝트의 역할

묵산 SaaS는 세 프로젝트가 결합되어 완성된다.

| 프로젝트 | 역할 |
|---|---|
| exam_system2 | 시험지, 문제, 해설, PDF, 수업자료 생성과 관리 |
| academy | 학생, 보호자, 반, 출결, 수납, 문자, 상담, 학원 운영 관리 |
| homepage | 공개 홈페이지, 상담 유입, 공지 노출, 결제 진입, 관리자 진입 |

각 프로젝트는 역할을 분명히 나누고, 장기적으로 API와 안정된 식별자로 연결한다.

---

## 3. 데이터 소유권 원칙

데이터 소유권은 다음을 원칙으로 한다.

| 데이터 | 소유 시스템 |
|---|---|
| 학생 | academy |
| 보호자 | academy |
| 반 | academy |
| 출결 | academy |
| 수납 | academy |
| 문자 발송 기록 | academy |
| 상담 기록 | academy |
| 시험지 | exam_system2 |
| 문항 | exam_system2 |
| 해설 | exam_system2 |
| PDF | exam_system2 |
| 수업자료 | exam_system2 |
| 홈페이지 공개 콘텐츠 | homepage |

홈페이지는 학생 개인정보, 결제 민감정보, 시험지 원본 데이터를 직접 소유하지 않는다.

---

## 4. Academy와의 연결

homepage는 academy와 다음 기능에서 연결된다.

### 상담 문의

홈페이지 상담 문의는 장기적으로 academy의 상담 또는 문의함으로 들어간다.

초기에는 홈페이지 DB 또는 임시 저장소에 보관할 수 있으나, 최종 목표는 academy 연결이다.

### 공지사항

공지사항은 초기에는 homepage에서 관리할 수 있다.

장기적으로는 academy의 공지·문자·알림 체계와 연결한다.

### 수강료 결제 진입

홈페이지는 결제를 직접 처리하지 않는다.

수강료 결제는 academy의 청구서, 결제 링크, PG 연동 흐름으로 연결한다.

홈페이지는 결제 링크 진입점만 제공한다.

### 관리자 진입

홈페이지는 academy 관리자 화면으로 들어가는 진입점을 제공한다.

---

## 5. Exam System과의 연결

homepage는 exam_system2와 직접 깊게 결합하지 않는다.

연결 가능 영역은 다음으로 제한한다.

- 시험지 생성관리 프로그램 소개
- 샘플 수업자료 소개
- 학원 내부 관리자 진입
- 필요 시 공개 가능한 샘플 PDF 또는 이미지 노출

금지 또는 보류:

- 시험지 원본 데이터 직접 노출
- 학생별 시험 결과 직접 노출
- 문제 DB 직접 검색
- 원문자료실 직접 공개
- 홈페이지에서 시험지 생성 실행

시험지 생성관리 기능은 관리자 영역에서 사용한다.

---

## 6. 식별자 원칙

장기 통합을 위해 안정된 식별자를 사용한다.

예상 식별자:

- academyId
- branchId
- homepageId
- inquiryId
- noticeId
- invoiceId
- paymentId
- resourceId
- testPaperId

homepage는 academyId를 기준으로 학원 홈페이지와 운영 데이터를 연결한다.

---

## 7. API 연결 원칙

프로젝트 간 연결은 직접 DB 공유가 아니라 API를 통해 수행한다.

금지:

- homepage가 academy DB를 직접 수정
- homepage가 exam_system2 DB를 직접 수정
- exam_system2가 homepage 파일을 직접 수정
- academy가 homepage 템플릿 파일을 직접 수정

허용:

- API를 통한 상담 문의 생성
- API를 통한 공지 목록 조회
- API를 통한 결제 링크 조회
- API를 통한 관리자 진입 링크 제공
- API를 통한 공개 샘플 자료 조회

---

## 8. 개인정보와 보안

홈페이지는 공개 영역이므로 개인정보 노출을 특히 조심한다.

홈페이지에 노출하면 안 되는 정보:

- 학생 실명
- 학생 성적
- 보호자 연락처
- 상담 내용
- 결제 내역
- 미납 여부
- 내부 수업자료 원본
- 비공개 강사 정보

상담 문의 입력 정보는 개인정보 수집 동의가 필요하다.

수강료 결제와 민감 정보는 academy와 PG 흐름에서 처리한다.

---

## 9. 초기 구현 원칙

초기에는 통합을 과도하게 구현하지 않는다.

우선순위:

1. 홈페이지 공개 콘텐츠 관리
2. 상담 문의 접수
3. 공지사항 기본 노출
4. 관리자 진입 링크
5. 결제 링크 진입 구조
6. academy 상담함 연동
7. academy 공지 연동
8. academy 결제 링크 연동
9. exam_system2 소개 또는 샘플 자료 연결

---

## 10. 한 줄 정의

homepage는 묵산 SaaS의 공개 접점이며, academy와 exam_system2를 외부 사용자에게 연결하되 각 시스템의 데이터 소유권과 경계를 침범하지 않는다.
