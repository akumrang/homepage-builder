# 첫 구현 체크리스트

Status: Draft  
Project: homepage  
Last Updated: 2026-06-24

---

## 1. 목적

이 문서는 홈페이지 MVP 첫 구현 후 반드시 확인해야 할 항목을 정의한다.

목표는 단순히 코드가 실행되는 것이 아니라, 묵산 홈페이지 프로젝트의 핵심 방향이 실제 구현에 반영되었는지 확인하는 것이다.

---

## 2. 실행 체크

- [ ] frontend dev 서버가 실행된다.
- [ ] backend dev 서버가 실행된다.
- [ ] frontend 포트는 5175를 사용한다.
- [ ] backend 포트는 4200을 사용한다.
- [ ] README에 실행 방법이 기록되어 있다.
- [ ] 실행 중 명백한 콘솔 오류가 없다.

---

## 3. 공개 홈페이지 체크

- [ ] `/h/sample-korean-academy` 경로로 접속할 수 있다.
- [ ] 샘플 학원명이 표시된다.
- [ ] 대표 문구가 표시된다.
- [ ] 대상 학년과 과목이 표시된다.
- [ ] 핵심 강점이 표시된다.
- [ ] 학원 소개가 표시된다.
- [ ] 강사진 정보가 표시된다.
- [ ] 커리큘럼 정보가 표시된다.
- [ ] 공지사항이 표시된다.
- [ ] 오시는 길 또는 주소가 표시된다.
- [ ] 상담 신청 버튼이 명확하게 보인다.

---

## 4. 모바일 품질 체크

- [ ] 모바일 폭에서 가로 스크롤이 생기지 않는다.
- [ ] 버튼이 손가락으로 누르기 쉬운 크기다.
- [ ] 글자가 너무 작지 않다.
- [ ] 메인 첫 화면에서 학원의 성격이 바로 보인다.
- [ ] 상담 신청까지 이동하기 쉽다.
- [ ] 레이아웃이 겹치거나 깨지지 않는다.

---

## 5. 상담 문의 체크

- [ ] 상담 문의 폼이 있다.
- [ ] 보호자 이름을 입력할 수 있다.
- [ ] 연락처를 입력할 수 있다.
- [ ] 학생 학년을 입력할 수 있다.
- [ ] 관심 과목을 입력할 수 있다.
- [ ] 문의 내용을 입력할 수 있다.
- [ ] 개인정보 동의 체크가 있다.
- [ ] 개인정보 동의 없이는 제출되지 않는다.
- [ ] 제출 시 backend API로 요청이 간다.
- [ ] backend에서 문의 내용을 확인할 수 있다.
- [ ] 제출 후 사용자에게 접수 완료 메시지가 보인다.

---

## 6. 내부 제작 화면 체크

- [ ] 내부 제작 화면 또는 관리자용 최소 화면이 있다.
- [ ] 샘플 학원 목록 또는 상태를 볼 수 있다.
- [ ] 접수된 문의 목록 또는 로그를 확인할 수 있다.
- [ ] 이 화면이 고객용 자유 편집기로 오해되지 않는다.

---

## 7. 데이터 경계 체크

- [ ] 실제 학생 개인정보를 사용하지 않았다.
- [ ] 실제 학부모 연락처를 사용하지 않았다.
- [ ] 실제 결제 정보를 사용하지 않았다.
- [ ] academy DB를 직접 연결하지 않았다.
- [ ] exam_system2 DB를 직접 연결하지 않았다.
- [ ] 다른 workspace 파일을 수정하지 않았다.
- [ ] 샘플 데이터임이 명확하다.

---

## 8. 품질 철학 체크

- [ ] 싸구려 자동 생성 사이트처럼 보이지 않는다.
- [ ] 과장 광고 문구가 없다.
- [ ] 허위 합격 실적이 없다.
- [ ] 허위 후기가 없다.
- [ ] 허위 강사 경력이 없다.
- [ ] 학부모가 보기에 신뢰감이 있다.
- [ ] 묵산의 고품질 원칙을 해치지 않는다.

---

## 9. 코드 품질 체크

- [ ] TypeScript 오류가 없다.
- [ ] build 또는 typecheck가 가능하다.
- [ ] frontend/backend 실행 스크립트가 명확하다.
- [ ] 샘플 데이터와 UI 코드가 지나치게 뒤섞이지 않았다.
- [ ] 상담 API 경로가 명확하다.
- [ ] 불필요한 대형 의존성을 추가하지 않았다.
- [ ] README와 실제 실행 방법이 일치한다.

---

## 10. Git 체크

- [ ] 변경 파일 목록이 명확하다.
- [ ] 불필요한 임시 파일이 없다.
- [ ] `.env` 또는 secret 파일이 커밋되지 않았다.
- [ ] 커밋 메시지가 작업 내용을 설명한다.
- [ ] `git status`가 clean이거나 남은 변경 사항을 명확히 설명한다.

---

## 11. 완료 판단

첫 구현은 다음이 만족되면 완료로 본다.

- 공개 홈페이지 1종이 실제로 보인다.
- 상담 문의가 접수된다.
- 내부 제작 화면에서 상태 또는 문의를 확인할 수 있다.
- 모바일에서 깨지지 않는다.
- academy와 exam_system2를 침범하지 않았다.
- README만 보고 실행할 수 있다.

---

## 12. 한 줄 기준

첫 구현의 성공 기준은 “많은 기능”이 아니라, 묵산 홈페이지 프로젝트의 핵심 구조가 작지만 품질 있게 실제로 작동하는 것이다.
