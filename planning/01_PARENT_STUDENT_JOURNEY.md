Status: Draft  
Project: homepage  
Last Updated: 2026-06-24

---

## 1. 목적

이 문서는 묵산 홈페이지 프로젝트에서 학부모와 학생이 학원을 어떻게 발견하고, 판단하고, 상담하고, 재원 후 서비스를 이용하는지를 정의한다.

홈페이지는 원장 만족용 장식물이 아니라, 학부모와 학생의 실제 판단 흐름을 기준으로 설계한다.

---

## 2. 핵심 사용자

| 사용자 | 상황 | 핵심 관심 |
|---|---|---|
| 예비 학부모 | 학원을 찾는 중 | 신뢰, 위치, 수업 대상, 상담 가능성 |
| 예비 학생 | 학원 분위기를 확인 | 수업 방식, 선생님, 부담감, 기대감 |
| 재원 학부모 | 이미 다니는 중 | 공지, 결제, 상담, 출결, 학원 소식 |
| 재원 학생 | 이미 다니는 중 | 과제, 자료, 시간표, 공지 |

## 3. 입학 전 여정

예비 학부모와 학생은 보통 다음 흐름으로 움직인다.

```text
검색
→ 첫 화면 확인
→ 학원 신뢰 판단
→ 수업 대상과 커리큘럼 확인
→ 위치와 시간표 확인
→ 강사진 또는 운영 방식 확인
→ 상담 문의
→ 방문 상담 또는 전화 상담
→ 등록 결정

4. 첫인상 판단 요소

학부모는 홈페이지에 들어온 뒤 짧은 시간 안에 다음을 판단한다.

이 학원이 우리 아이 학년에 맞는가?
어떤 과목에 강한가?
내신 중심인가, 수능 중심인가, 초중등 기초 중심인가?
선생님이 믿을 만한가?
학원이 실제로 운영되고 있는 느낌이 있는가?
위치와 수업 시간이 맞는가?
상담 신청이 쉬운가?
과장 광고가 아니라 신뢰감이 있는가?

6. 상담 전환 흐름

상담 신청은 복잡하면 안 된다.

기본 상담 흐름은 다음을 원칙으로 한다.

상담 신청 버튼
→ 이름, 연락처, 학생 학년, 관심 과목 입력
→ 개인정보 동의
→ 신청 완료
→ academy 상담함으로 접수
→ 원장 또는 담당자가 확인

상담 신청 완료 후에는 학부모가 안심할 수 있는 안내가 필요하다.

예:

상담 신청이 접수되었습니다. 학원에서 확인 후 연락드리겠습니다.

7. 재원 후 여정

재원 후 학부모는 홈페이지를 매일 둘러보지는 않는다.

재원 후 핵심 접점은 모바일 웹 또는 향후 학부모 앱이 된다.

문자 또는 알림 수신
→ 링크 클릭
→ 모바일 화면 접속
→ 공지 확인
→ 수강료 결제
→ 필요 시 상담 문의
8. 재원 학부모가 원하는 기능
공지 확인
수강료 결제
결제 내역 확인
출결 또는 등하원 알림
시간표 확인
상담 문의
학원 자료 또는 안내문 확인

초기 홈페이지 프로젝트에서는 이 전체를 구현하지 않는다.

다만 홈페이지 구조는 향후 학부모 모바일 화면으로 확장될 수 있어야 한다.

9. 설계 원칙

홈페이지는 입학 전 설득 창구다.

재원 후 기능은 모바일 웹 또는 학부모 앱으로 발전할 수 있는 접점으로 설계한다.

따라서 홈페이지는 다음을 우선한다.

신뢰감
빠른 정보 파악
쉬운 상담 신청
모바일 가독성
학원관리 시스템과의 연결성
10. 한 줄 정의

묵산 홈페이지는 입학 전에는 학원의 첫인상을 만들고, 입학 후에는 학부모 모바일 접점으로 이어지는 문이다.
