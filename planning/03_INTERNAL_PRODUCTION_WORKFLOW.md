# 홈페이지 내부 제작 공정

Status: Draft  
Project: homepage  
Last Updated: 2026-06-24

---

## 1. 목적

이 문서는 묵산 홈페이지 프로젝트에서 학원당 1개의 홈페이지를 제작·검수·게시·관리하는 내부 공정을 정의한다.

묵산은 고객에게 홈페이지 제작툴을 판매하지 않는다.

묵산 내부 제작 시스템과 표준 공정을 통해 고품질 홈페이지를 제공한다.

---

## 2. 핵심 원칙

홈페이지 제작은 수작업 예술품 제작이 아니라, 고품질을 반복 가능하게 만드는 표준 공정이어야 한다.

다만 자동화는 싸구려 대량생산을 뜻하지 않는다.

자동화의 목적은 다음이다.

- 품질 기준 유지
- 제작 시간 단축
- 반복 업무 감소
- 1인 또는 극소수 운영 가능성 확보
- 누적 3천~1만 개 홈페이지 관리 가능성 확보

## 3. 전체 제작 흐름

기본 흐름은 다음과 같다.

```text
가입 또는 제작 요청
→ 학원 정보 수집
→ 자료 누락 확인
→ AI 초안 생성
→ 템플릿 선택 또는 추천
→ 홈페이지 초안 생성
→ 내부 검수
→ 고객 확인
→ 수정 반영
→ 게시
→ 운영 관리

4. 제작 상태

각 홈페이지 제작 건은 상태를 가진다.

상태	의미
REQUESTED	제작 요청 접수
WAITING_FOR_MATERIALS	학원 자료 대기
MATERIALS_READY	제작 자료 준비 완료
AI_DRAFTED	AI 초안 생성 완료
TEMPLATE_SELECTED	템플릿 선택 완료
DRAFT_CREATED	홈페이지 초안 생성 완료
INTERNAL_REVIEW	묵산 내부 검수 중
CUSTOMER_REVIEW	고객 확인 중
REVISION_REQUESTED	수정 요청 접수
APPROVED	고객 승인 완료
PUBLISHED	게시 완료
ARCHIVED	사용 종료 또는 보관

초기 화면에서는 영어 상태명을 그대로 노출하지 않는다.

고객 화면에는 다음처럼 표현한다.

초안 제작 중
검수 중
확인 요청

게시 완료

6. 자료 누락 확인

자료가 부족하면 제작을 바로 진행하지 않는다.

시스템은 누락된 자료를 자동으로 표시해야 한다.

예:

로고 없음
대표 사진 없음
강사진 정보 부족
커리큘럼 설명 부족
상담 연락처 없음
주소 없음

초기에는 사람이 확인하더라도, 장기적으로는 자동 체크리스트화한다.

7. AI 초안 생성

AI는 학원 정보를 바탕으로 다음 초안을 만든다.

메인 헤드라인
학원 소개
핵심 강점 3개
커리큘럼 설명
강사진 소개 문구
상담 유도 문구
FAQ
SEO 제목
SEO 설명

AI 초안은 최종본이 아니다.

반드시 내부 검수와 고객 확인을 거친다.

8. 템플릿 선택

초기에는 자유 디자인이 아니라 제한된 템플릿을 사용한다.

템플릿 선택 기준:

학원 과목
대상 학년
학원 규모
학원 분위기
입시 중심 여부
초중등 중심 여부
고등 전문 여부

초기 템플릿 수는 3~5개 정도로 제한한다.

9. 내부 검수

묵산 내부 검수는 필수다.

검수 항목:

허위 또는 과장 표현 여부
AI가 지어낸 실적 여부
강사 경력 사실성
상담 버튼 정상 작동
모바일 화면 가독성
오타
지도와 연락처 정확성
개인정보 동의 문구 존재 여부
공지와 결제 링크 노출 방식
전체 품질감
10. 고객 확인

고객은 홈페이지 초안을 확인하고 제한된 범위 안에서 수정 요청을 할 수 있다.

기본 수정 허용 범위:

오타 수정
연락처 수정
주소 수정
강사진 정보 수정
커리큘럼 문구 수정
사진 교체
상담 시간 수정

초기 계약 또는 운영 정책에서 무제한 디자인 수정은 허용하지 않는다.

11. 게시

고객 승인 후 홈페이지를 게시한다.

게시 전 확인:

도메인 또는 서브도메인 연결
HTTPS 적용
모바일 확인
상담 문의 테스트
공지 노출 확인
관리자 진입 링크 확인
백업 또는 배포 기록 저장
12. 운영 관리

게시 후에도 홈페이지는 살아 있는 자산이다.

다만 유지관리는 제한된 구조로 운영한다.

기본 관리 항목:

연락처 변경
시간표 변경
강사진 변경
커리큘럼 변경
공지 등록
사진 교체
상담 문의 확인

디자인 전체 변경, 대규모 구조 변경, 별도 페이지 추가는 유료 또는 별도 정책으로 분리한다.

13. 자동화 우선순위

자동화는 다음 순서로 진행한다.

자료 수집 양식
자료 누락 체크
AI 문구 초안 생성
템플릿 자동 반영
미리보기 생성
고객 확인 링크
게시 자동화
장애 감시
반복 문의 자동 응답
제작 상태 대시보드
14. 실패와 예외 처리

모든 제작 건은 예외가 생길 수 있다.

예외 유형:

고객 자료 미제출
사진 품질 부족
허위 경력 의심
과도한 수정 요청
도메인 연결 지연
상담폼 테스트 실패
모바일 화면 깨짐

예외는 상태와 메모로 기록한다.

묵산은 예외만 판단하고, 반복 확인은 시스템이 맡는다.

15. 한 줄 정의

묵산 홈페이지 제작 공정은 1인 또는 극소수 운영자가 고품질 홈페이지를 반복적으로 제작·검수·게시할 수 있게 하는 내부 생산 시스템이다.
