Status: Draft  
Project: homepage  
Last Updated: 2026-06-29

---

## 1. 목적

이 문서는 완성형 샘플 갤러리를 구현하기 전에, 묵산 내부 상담/검토용으로 사용할 정적 샘플 자료의 구성안을 정의한다.

현재 목표는 구현이 아니다. `trust-basic-v1` 실제 샘플, 샘플 방향 3종 브리프, 샘플 카드 문안, 고객 기대 관리 문구를 하나의 검토 흐름으로 묶어 묵산이 고객 상담 또는 내부 판정에서 일관되게 사용할 수 있게 하는 것이다.

이 자료는 고객용 자유형 홈페이지 제작툴이 아니며, 공개 `/samples` 또는 `/gallery` 라우트 구현을 의미하지 않는다.

---

## 2. 자료의 성격

정적 샘플 자료는 다음 성격을 가진다.

| 항목 | 기준 |
|---|---|
| 주 사용자 | 묵산 내부 제작자 또는 상담자 |
| 고객 노출 | 상담 중 제한적으로 일부 화면만 공유 가능 |
| 형태 | 문서, PDF, 슬라이드, 캡처 묶음 중 하나 |
| 목적 | 고객 선호 방향 확인과 내부 제작 판단 |
| 데이터 | 샘플 데이터와 브리프만 사용 |
| 금지 | 고객 선택 제출 API, 고객 포털, 자유 편집 UI, 즉시 게시 흐름 |

정적 샘플 자료는 제품 구현물이 아니라 운영 자료다. 고객이 이 자료를 본다고 해서 샘플 선택 기능이나 갤러리 기능이 출시된 것으로 보이면 안 된다.

---

## 3. 구성 원칙

자료는 다음 순서로 읽히게 구성한다.

1. 묵산 홈페이지 제작 방식 안내
2. 고객이 선택할 수 있는 권리와 선택할 수 없는 권리 구분
3. 샘플 방향 3종 한눈 비교
4. 현재 실제 구현된 `trust-basic-v1` 샘플 확인
5. 아직 브리프 단계인 후보 방향 확인
6. 고객 자료 준비 항목 확인
7. 묵산 내부 제작자 판정 메모
8. 다음 액션 결정

핵심 원칙:

- 구현된 것과 브리프 단계인 것을 명확히 구분한다.
- 고객에게 보여도 되는 문구와 내부 전용 메모를 분리한다.
- 샘플 선택이 자동 적용 또는 즉시 게시처럼 보이지 않게 한다.
- 허위 실적, 허위 후기, 허위 강사 경력을 유도하는 문구를 넣지 않는다.
- 고객이 “저 사이트처럼 똑같이” 요구하기 쉬운 표현을 피한다.

## 4. 권장 자료 구조

### 4.1 표지

목적:

- 자료가 샘플 방향 확인용이라는 점을 먼저 고정한다.

표지 문구:

```text
묵산 학원 홈페이지 샘플 방향 검토 자료

이 자료는 홈페이지 제작 방향을 검토하기 위한 내부/상담용 자료입니다.
최종 홈페이지는 학원 자료 확인, 묵산 제작, 내부 검수, 고객 확인 절차를 거쳐 완성됩니다.
```

표지에 넣지 않을 것:

- “직접 제작”
- “즉시 생성”
- “원하는 대로 편집”
- “사이트 복제”

### 4.2 제작 방식 안내

넣을 문구:

```text
고객은 자유형 홈페이지 빌더를 사용하는 것이 아니라, 묵산이 준비한 완성형 방향 중 선호를 제공합니다.
묵산 내부 제작자는 학원 자료와 선호 방향을 바탕으로 표준 템플릿 안에서 홈페이지를 제작하고 검수합니다.
```

포함할 흐름:

```text
샘플 방향 확인
→ 고객 선호 방향 메모
→ 학원 자료 수집
→ 묵산 내부 제작
→ 묵산 검수
→ 고객 확인
→ 게시
```

### 4.3 고객 권리와 제한

고객에게 줄 권리:

- 학원명, 로고, 대표 문구 제공
- 대표 색상 선호 제시
- 사진 또는 사진 방향 제시
- 분위기 선택: 신뢰형, 입시형, 소수정예형
- 강조 장점 제시
- 상담 문구 톤 선택
- 최종 게시 전 확인과 수정 요청

고객에게 주지 않을 권리:

- 레이아웃 무제한 변경
- 섹션 자유 조립
- CSS/코드 직접 수정
- 고객별 별도 코드베이스 요구
- 타 사이트 복제 요구
- 검수 없는 즉시 게시

### 4.4 샘플 방향 3종 요약

한눈 비교 표:

| 방향 | 현재 상태 | 적합한 학원 | 주의점 |
|---|---|---|---|
| 신뢰형 기본 `trust-basic-v1` | 구현됨 | 국어, 영어, 보습, 중소형 단과 | 가장 안전한 첫 파일럿 방향 |
| 내신/입시 관리형 `exam-focus-v1` | 브리프 단계 | 고등 내신, 수능, 시험 대비 | 실적/성과 문구 검수 필요 |
| 소수정예 관리형 `small-group-v1` | 브리프 단계 | 작은 반, 과제/오답 관리, 학부모 소통 | 실제 관리 루틴 자료 필요 |

고객에게 보여줄 때는 `현재 상태` 대신 다음 표현을 쓴다.

| 내부 표현 | 고객용 표현 |
|---|---|
| 구현됨 | 실제 샘플 확인 가능 |
| 브리프 단계 | 방향 예시 |
| 검수 리스크 | 확인 자료가 필요 |

### 4.5 실제 샘플 확인: `trust-basic-v1`

현재 실제 화면으로 확인할 수 있는 샘플:

```text
/h/sample-korean-academy
```

확인할 장면:

1. 첫 화면: 학원명, 대표 문구, 상담 CTA
2. 핵심 강점: 수업 후 점검, 학년별 흐름, 보호자 소통
3. 소개: 철학, 운영, 관리
4. 강사진: 확인 가능한 역할과 수업 초점 중심 소개
5. 커리큘럼: 학년별 수업 흐름
6. 공지: 상담/수업 안내
7. 오시는 길과 상담 신청

캡처 기준:

- desktop 첫 화면
- mobile 첫 화면
- 상담 폼 영역
- curriculum 또는 소개 영역
- footer 샘플 고지

사용 가능한 근거:

- `docs/24_SCREENSHOT_REGRESSION_TEST_REPORT.md`
- `npm.cmd run visual:regression`
- 로컬 또는 리허설 환경의 `/h/sample-korean-academy`

주의:

- `.tmp/visual-regression/*` 캡처 산출물은 Git에 커밋하지 않는다.
- 고객에게 보여줄 경우 샘플 홈페이지이며 실제 개인정보가 없다는 고지를 유지한다.

### 4.6 브리프 단계 방향 확인

`exam-focus-v1`과 `small-group-v1`은 아직 실제 구현 화면이 아니다.

자료에는 다음처럼 표시한다.

```text
이 방향은 현재 브리프 단계입니다.
실제 화면 구현 전이며, 고객 자료와 검수 기준이 충분할 때 제작 후보로 검토합니다.
```

내신/입시 관리형에서 강조할 것:

- 학년별 대비 과정
- 시험 대비 자료 관리
- 과제/오답 피드백 루틴
- 성과 문구는 확인 자료가 있을 때만 사용

소수정예 관리형에서 강조할 것:

- 작은 반 운영
- 과제 점검
- 학부모 소통
- 실제 관리 루틴이 있어야 설득력 있음

### 4.7 고객 자료 준비 체크

자료 말미에 다음 체크를 둔다.

| 항목 | 필요 여부 | 비고 |
|---|---:|---|
| 학원명 | 필수 | 공식 표기 |
| 로고 | 권장 | 없으면 텍스트 로고 승인 필요 |
| 대표 문구 | 필수 | 1문장 |
| 대표 색상 선호 | 선택 | 묵산 디자인 토큰 안에서 조정 |
| 사진 | 권장 | 실제 학생 식별 금지 |
| 강사진 소개 | 권장 | 확인 가능한 역할과 수업 초점 |
| 커리큘럼 | 필수 | 학년/과목별 |
| 공지 | 권장 | 개강, 상담, 시험 대비 |
| 주소/전화/운영 시간 | 필수 | 공개 가능 정보 |
| 실적/후기 | 선택 | 확인 자료 없으면 사용 금지 |

### 4.8 내부 제작자 판정 메모

상담 또는 검토 후 내부 제작자는 다음 양식으로 기록한다.

```text
검토 일시:
고객/학원명:
상담자:

고객 선호 방향:
1순위:
2순위:

선택 이유:

자료 충분성:
- 로고:
- 사진:
- 강사진:
- 커리큘럼:
- 공지:
- 위치/연락처:
- 실적/후기 근거:

기대 관리 필요 항목:

추천 제작 방향:

다음 요청 자료:

GO / HOLD / NO-GO:
```

판정 기준:

- `GO`: 기본 자료가 충분하고 기대 관리 리스크가 낮다.
- `HOLD`: 추가 자료가 필요하다.
- `NO-GO`: 허위 실적, 개인정보, 무한 맞춤 요구, 외부 사이트 복제 요구가 있다.

## 5. 자료 구분

같은 내용을 두 버전으로 나누는 것을 권장한다.

### 5.1 내부용 전체본

포함:

- 샘플 방향 3종 비교
- 구현/브리프 상태 구분

- 고객 기대 관리 문구
- 내부 판정 메모
- GO/HOLD/NO-GO 기준

고객에게 그대로 전달하지 않는다.

### 5.2 상담 공유용 축약본

포함:

- 전체 안내 문구

- 자료 준비 체크 일부

제외:

- 내부 검수 리스크 표현
- 구현 우선순위
- 내부 판정 메모
- 운영 난이도
- 접근 제어, DB, 배포 관련 내용

## 7. 사용 순서

묵산 내부 상담에서 권장 순서는 다음이다.

1. 고객에게 제작 방식 안내
2. 전체 안내 문구로 기대 관리
3. 세 방향 카드 비교
4. `trust-basic-v1` 실제 샘플 화면 확인
5. 고객이 가까운 방향 1~2개 선택
6. 필요한 자료 체크
7. 내부 제작자가 GO/HOLD/NO-GO 판정
8. 고객에게 추가 자료 요청 또는 제작 방향 안내

상담 중 고객에게 보여도 되는 말:

```text
지금은 방향을 고르는 단계입니다.
실제 제작은 학원 자료와 검수 기준을 확인한 뒤 진행됩니다.
```

내부자가 피해야 할 말:

```text
이걸 고르면 그대로 만들어드립니다.
원하시는 대로 마음껏 바꿀 수 있습니다.
다른 사이트처럼 똑같이 해드립니다.
```

## 8. 이번 구성안의 결론

정적 샘플 자료는 샘플 갤러리 구현 전 단계의 운영 안전장치다.

이 자료를 먼저 만들면 다음을 확인할 수 있다.

- 고객이 방향 선택을 이해하는지
- 자유형 빌더로 오해하는 지점이 있는지
- `trust-basic-v1` 하나만으로 상담이 가능한지
- 브리프 단계 후보를 보여줄 때 기대가 과도해지는지
- 어떤 자료가 자주 누락되는지

따라서 다음 구현으로 바로 넘어가지 않고, 먼저 정적 자료 기준으로 내부 검토와 상담 리허설을 진행하는 것이 안전하다.
