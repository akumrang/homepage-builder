# MVP 이후 우선순위 재정렬

Status: Draft  
Project: homepage  
Last Updated: 2026-06-26

---

## 1. 목적

이 문서는 홈페이지 MVP 첫 구현 이후의 실제 개발 우선순위를 다시 정리한다.

특히 `완성형 샘플 갤러리`가 제품 전략상 중요한 후속 후보로 떠올랐지만, 현재 MVP 다음 즉시 구현 대상은 아니라는 판단을 전제로 한다.

---

## 2. 현재 상태

완료된 핵심 작업:

- `trust-basic-v1` 공개 홈페이지 1종 구현
- `/h/sample-korean-academy` 렌더링
- 상담 문의 폼과 backend 저장
- `/internal` 내부 제작 화면
- 공지 CRUD와 문의 상태 관리
- 콘텐츠 점검 API
- shared 문의 검증 규칙
- 브라우저 기반 1차 시각 QA
- 제품 정의와 고객 디자인 권리 원칙 문서화

보류로 확정한 것:

- 완성형 샘플 갤러리 실제 구현
- 고객의 샘플 선택 제출 기능
- 고객 또는 운영자의 제한 편집 기능
- 다중 템플릿 전체 구현
- academy, exam_system2 실제 연동

---

## 3. 우선순위 판단 기준

MVP 이후의 다음 개발은 다음 기준으로 고른다.

1. 외부 노출 시 위험을 줄이는가?
2. 묵산 1인 또는 극소수 운영 가능성을 높이는가?
3. 고객용 자유형 빌더로 오해될 가능성을 낮추는가?
4. 현재 MVP 구조를 안정화하는가?
5. 샘플 갤러리나 다중 템플릿처럼 범위가 커지는 기능보다 앞서야 하는가?

## 4. 후보별 판단

| 후보 | 판단 | 이유 |
|---|---|---|
| 내부 제작 화면 접근 경계 MVP | 1순위 | `/internal`은 문의와 제작 상태를 다루므로 외부 노출 전 최소 접근 경계가 필요하다. 전체 로그인·권한 구현은 아니어도 내부용 보호 장치는 먼저 필요하다. |
| 2차 시각 QA | 2순위 | 1차 QA는 통과했지만 상담 폼 오류/완료 상태, 내부 탭 전환, 전체 하단 스크롤 확인이 남아 있다. |
| 자료 누락 체크와 제작 준비도 강화 | 3순위 | 내부 제작 공정 자동화의 첫 실무 가치가 크다. 다만 접근 경계 이후가 안전하다. |
| Prisma migration과 배포 환경 정리 | 4순위 | 운영 전 필요하지만, 현재 로컬 MVP 단계에서는 접근 경계와 QA 이후에 진행해도 된다. |
| 완성형 샘플 갤러리 기획 | 후속 후보 | 고객 디자인 권리 2단계인 `방향 선택형`의 후보지만, 즉시 구현 대상은 아니다. |
| 완성형 샘플 갤러리 구현 | 보류 | 템플릿 수, 공개 범위, 고객 선택 제출 여부, 커스터마이징 기대 관리가 정리된 뒤 진행한다. |
| academy/exam_system2 실제 연동 | 보류 | MVP 이후에도 API 경계 설계가 먼저이며 직접 DB 공유는 금지한다. |

## 5. 다음 1순위 구현 후보

다음 실제 구현 후보는 다음으로 정한다.

```text
내부 제작 화면 접근 경계 MVP
```

목표:

- `/internal`이 공개 홈페이지처럼 아무나 접근 가능한 화면으로 남지 않게 한다.
- 상담 문의 목록, 공지 관리, 제작 상태 변경 API를 내부용 API로 구분한다.
- 전체 로그인·권한 시스템은 만들지 않는다.
- 고객용 자유형 빌더, 고객 계정, 역할 기반 권한은 만들지 않는다.

가능한 최소 구현 범위:

- backend에서 내부 API와 공개 API를 구분한다.
- 내부 API는 개발용 internal access token 또는 유사한 최소 경계를 요구한다.
- public homepage와 상담 문의 제출은 계속 공개로 둔다.
- frontend `/internal`은 내부 접근 키 입력 또는 설정된 내부 토큰이 없으면 대시보드를 노출하지 않는다.
- README에 로컬 개발용 내부 접근 방법을 기록한다.
- smoke test에 내부 API 보호 확인을 추가한다.

이번 후보에서 하지 않을 것:

- 사용자 계정 생성
- 로그인/회원가입 화면
- 역할 기반 권한 전체 구현
- 고객 포털
- OAuth 또는 외부 인증 연동
- 운영용 보안 완성본 선언

## 6. 1순위 후보의 성공 기준

다음이 만족되면 `내부 제작 화면 접근 경계 MVP`는 완료로 본다.

- 공개 홈페이지 `/h/sample-korean-academy`는 토큰 없이 접근 가능하다.
- 상담 문의 `POST /api/inquiries`는 토큰 없이 제출 가능하다.
- 문의 목록 조회, 문의 상태 변경, 공지 CRUD, 제작 상태 변경 등 내부 API는 보호된다.
- `/internal`은 내부 접근 경계 없이 운영 데이터를 바로 보여주지 않는다.
- `npm.cmd run verify`가 통과한다.
- README가 내부 접근 방법과 한계를 설명한다.

## 7. 샘플 갤러리의 현재 지위

샘플 갤러리는 삭제된 아이디어가 아니다.

다만 현재 지위는 다음이다.

```text
샘플 갤러리 = 고객 디자인 권리 2단계 후보
현재 MVP 구현 범위 = 아님
다음 즉시 구현 대상 = 아님
먼저 기획 검토 필요
```

샘플 갤러리는 최소 2~3개 이상의 실제 템플릿 방향, 고객 기대 관리 문구, 선택값 처리 방식이 정리된 뒤 구현 여부를 다시 판단한다.

---

## 8. 결론

샘플 갤러리를 바로 만들면 제품 방향은 넓어지지만, 현재 MVP의 운영 안정성은 높아지지 않는다.

따라서 현재 우선순위는 다음 순서로 둔다.

1. 내부 제작 화면 접근 경계 MVP
2. 2차 시각 QA
3. 자료 누락 체크와 제작 준비도 강화
4. Prisma migration과 배포 환경 정리
5. 샘플 갤러리 기획 재검토
