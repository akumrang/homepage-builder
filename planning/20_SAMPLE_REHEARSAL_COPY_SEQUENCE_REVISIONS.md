## 1. 목적

이 문서는 `planning/19_VIRTUAL_CUSTOMER_SAMPLE_REHEARSAL_RECORD.md`에서 확인된 보완점을 반영해 고객 공유용 문구와 자료 순서를 다듬은 기록이다.

현재 목표는 구현이 아니다. 묵산이 베틀 시스템 외부 브랜드 문구 톤을 확인할 수 있도록 고객에게 보일 가능성이 있는 표현을 더 부드럽고 명확하게 정리한다.

---

## 2. 보완 원칙

이번 보완은 다음 원칙을 따른다.

- 고객의 선택권은 인정한다.
- 자유형 빌더로 오해될 표현은 줄인다.
- “제한”이라는 단어보다 “제작 안정성”과 “베틀 시스템 검수”를 앞세운다.
- 내부 용어인 “표준 템플릿”은 고객용 문구에서 필요할 때만 쓴다.
- 샘플이 실제 개인정보 없는 참고 자료임을 시작과 끝에서 모두 알린다.

## 3. 고객 공유용 전체 안내 문구 보완

기존 문구:

```text
아래 샘플은 학원 홈페이지의 제작 방향을 고르기 위한 참고 화면입니다.
선택하신 방향은 최종 디자인 확정이 아니며, 학원 자료 확인과 베틀 시스템 내부 검수 후 제작됩니다.
레이아웃 자유 편집, 코드 수정, 타 사이트 복제 방식의 맞춤 제작은 제공하지 않습니다.
```

보완 문구:

```text
아래 샘플은 우리 학원에 어울리는 홈페이지 방향을 함께 고르기 위한 참고 자료입니다.
선택하신 방향은 최종 디자인 확정이 아니라 제작을 시작하기 위한 기준점입니다.
최종 홈페이지는 학원 자료 확인, 베틀 시스템 제작, 내부 검수, 고객 확인을 거쳐 완성됩니다.
```

별도 고지 문구:

```text
제작 안정성을 위해 레이아웃 자유 편집, 코드 수정, 타 사이트 복제 방식의 제작은 제공하지 않습니다.
```

판단:

- 첫 문장에서는 부드러운 표현을 쓴다.
- 금지 항목은 한 문단 아래 별도 고지로 분리해 방어적으로 들리는 느낌을 줄인다.

## 4. 상담 구두 설명 보완

기존 문구:

```text
샘플 중 마음에 가까운 방향을 골라주시면, 그 방향을 기준으로 학원 자료와 운영 방식에 맞게 베틀 시스템이 제작합니다.
샘플을 그대로 복사하거나 고객이 직접 편집하는 방식은 아닙니다.
```

보완 문구:

```text
샘플 중 우리 학원과 가장 가까운 분위기를 골라주시면, 베틀 시스템이 그 방향을 참고해 학원 자료와 운영 방식에 맞게 제작합니다.
샘플을 그대로 복사하는 방식은 아니며, 고객님이 직접 편집하는 제작툴도 아닙니다.
```

더 짧은 상담 문구:

```text
오늘은 완성 디자인을 확정하기보다, 학원에 맞는 제작 방향을 고르는 단계로 봐주시면 됩니다.
```

## 5. 카드 문구 보완

### 5.1 신뢰형 기본

기존 핵심 문구:

```text
과장 없이 학원의 수업 흐름과 관리 방식을 차분하게 보여주는 기본 방향입니다.
자료가 많지 않은 중소형 단과 학원에도 안정적으로 적용할 수 있습니다.
```

보완 문구:

```text
학원의 수업 흐름과 관리 방식을 차분하게 보여주는 방향입니다.
처음 홈페이지를 준비하거나 자료가 많지 않은 학원도 부담 없이 검토할 수 있습니다.
```

상담용 한 줄:

```text
첫 홈페이지 제작에는 이 방향이 가장 무난하고 안전합니다.
```

톤 확인 필요:

- “무난하고 안전합니다”가 너무 평범하게 들릴 수 있다.
- 묵산 확인 후 “차분하고 안정적입니다”로 바꿀지 판단한다.

### 5.2 내신/입시 관리형

기존 핵심 문구:

```text
시험 대비 과정, 학년별 커리큘럼, 오답 관리 흐름을 선명하게 보여주는 방향입니다.
성과 문구는 확인 가능한 자료가 있을 때만 사용합니다.
```

보완 문구:

```text
시험 대비 과정, 학년별 커리큘럼, 오답 관리 흐름을 또렷하게 보여주는 방향입니다.
합격 실적이나 성과 표현은 확인 가능한 자료가 있을 때만 반영합니다.
```

상담용 한 줄:

```text
시험 대비 체계가 분명한 학원에 잘 맞지만, 성과 표현은 검수 기준이 가장 엄격합니다.
```

톤 확인 필요:

- “가장 엄격합니다”가 고객에게 차갑게 들릴 수 있다.
- 고객용으로는 “자료 확인이 꼭 필요합니다”가 더 부드럽다.

### 5.3 소수정예 관리형

기존 핵심 문구:

```text
작은 반, 과제 점검, 학부모 소통처럼 가까운 관리 방식을 보여주는 방향입니다.
실제 운영 루틴이 있을 때 가장 설득력이 좋습니다.
```

보완 문구:

```text
작은 반, 과제 점검, 학부모 소통처럼 가까운 관리 방식을 보여주는 방향입니다.
실제 관리 루틴이 정리되어 있을수록 학원의 장점이 선명하게 드러납니다.
```

상담용 한 줄:

```text
작은 반과 꾸준한 피드백이 실제 강점인 학원에 잘 맞습니다.
```

톤 확인 필요:

- 현재 문구는 비교적 자연스럽다.
- “가까운 관리”가 베틀 시스템 외부 브랜드 톤에 맞는지 확인하면 된다.

## 6. FAQ 보완

보완 답변:

```text
샘플은 제작 방향을 정하기 위한 참고 자료입니다.
학원 자료와 검수 기준에 따라 세부 구성은 조정되며, 최종 게시 전 고객 확인 절차를 거칩니다.
```

### 질문: 샘플끼리 섞을 수 있나요?

보완 답변:

```text
원하시는 강조점은 참고할 수 있습니다.
다만 섹션을 자유롭게 조립하는 방식은 아니며, 베틀 시스템이 운영 가능한 표준 구조 안에서 제작합니다.
```

### 질문: 다른 학원 사이트와 비슷하게 만들 수 있나요?

보완 답변:

```text
참고하고 싶은 분위기는 들을 수 있습니다.
다만 타 사이트를 복제하지는 않으며, 베틀 시스템의 표준 제작 방식 안에서 새로 구성합니다.
```

### 질문: 선택하면 바로 게시되나요?

보완 답변:

```text
아닙니다.
선택은 제작 방향을 정하기 위한 단계이며, 베틀 시스템 제작과 내부 검수, 고객 확인을 거친 뒤 게시됩니다.
```

## 7. 자료 순서 보완

기존 권장 순서:

1. 고객에게 제작 방식 안내
2. 전체 안내 문구로 기대 관리
3. 세 방향 카드 비교
4. `trust-basic-v1` 실제 샘플 화면 확인
5. 고객이 가까운 방향 1~2개 선택
6. 필요한 자료 체크
7. 내부 제작자가 GO/HOLD/NO-GO 판정
8. 고객에게 추가 자료 요청 또는 제작 방향 안내

보완 순서:

1. 제작 방식 안내
2. 샘플이 실제 개인정보 없는 참고 자료임을 고지
3. 고객이 선택할 수 있는 것 안내
4. 제작 안정성을 위해 제공하지 않는 것 안내
5. 세 방향 카드 비교
6. `trust-basic-v1` 실제 샘플 화면 확인
7. 후보 방향은 브리프 단계임을 안내
8. 고객 선호 방향 1~2개 메모
9. 자료 준비 체크
10. 내부 GO/HOLD/NO-GO 판정
11. 고객에게 추가 자료 또는 제작 방향 안내

보완 이유:

- 고객 권리를 먼저 보여준 뒤 제한을 설명하면 방어적으로 덜 들린다.
- 실제 샘플 확인 전에 샘플 고지를 먼저 넣어 오해를 줄인다.
- 브리프 단계 후보는 실제 화면 확인 이후에 설명하는 편이 기대를 낮춘다.

## 8. 문구 톤 확인 요청 대상

### 8.1 2026-06-30 묵산 HOLD 판정 반영

판정: `HOLD`

사유:

- `묵산`은 개발 대화와 내부 판단에서 사용하는 호칭이다.
- 고객에게 노출되는 외부 브랜드명은 `베틀 시스템`이다.
- 고객용 화면, 발송 문안, 샘플 설명, 상담 자료에는 `묵산`을 브랜드명처럼 쓰지 않는다.

반영:

- 고객에게 보일 수 있는 제작 주체 표기를 `베틀 시스템`으로 교정했다.
- `README.md`와 `docs/00_HOMEPAGE_PROJECT_CONSTITUTION.md`에 외부 브랜드 표기 원칙을 추가했다.
- 코드 enum 값인 `MUKSAN_CREATED`, `MUKSAN_APPROVED_REPLACEMENT`는 저장 데이터 식별자이므로 이번 작업에서는 유지하고, 화면 라벨만 `베틀 시스템 제작`, `베틀 시스템 승인 대체`로 교정했다.

---

묵산 확인이 필요한 문구:

1. “우리 학원에 어울리는 홈페이지 방향”
2. “제작을 시작하기 위한 기준점”
3. “제작 안정성을 위해”
4. “처음 홈페이지를 준비하거나 자료가 많지 않은 학원도 부담 없이 검토”
5. “성과 표현은 확인 가능한 자료가 있을 때만 반영”
6. “실제 관리 루틴이 정리되어 있을수록 학원의 장점이 선명하게 드러납니다”

확인 관점:

- 너무 영업적으로 들리지 않는가
- 너무 딱딱하거나 방어적으로 들리지 않는가
- 고객에게 권리를 주되 무한 맞춤 기대를 만들지 않는가
- 베틀 시스템 외부 브랜드 톤과 맞는가

## 9. 이번 보완의 결론

문구의 기본 방향은 유지하되, 고객이 처음 접하는 문장은 더 부드럽게 바꾸는 것이 좋다.

가장 중요한 변화는 다음이다.

```text
제한을 먼저 말하지 않고,
고객이 선택할 수 있는 것과 베틀 시스템이 완성해주는 방식을 먼저 설명한다.
```

다만 금지 범위는 약화하지 않는다. 자유형 편집, 코드 수정, 타 사이트 복제, 검수 없는 게시 금지는 그대로 유지한다.

## 10. 다음 작업

다음 1순위 추천 작업은 베틀 시스템 외부 브랜드 문구 톤 재확인이다.

묵산 확인 후 다음 중 하나로 진행한다.

- GO: 보완 문구를 고객 공유용 문서에 반영
- HOLD: 특정 표현만 재수정
- NO-GO: 고객 공유용 샘플 자료 구성 전면 재검토
